## Design Decisions (Review / Override)

These choices were made to maximize character-package feature coverage. They're reasonable defaults but may not match the author's vision.

### Hidden Relationships
- [ ] **Viola as half-sister** — Chose her over Chelsea because "actress with hidden identity" ties into performance/deception themes. Is this the right pick?
- [ ] **Chelsea as possible daughter** — Gave her ambiguity (suspects but doesn't know for sure). Should this be definitive?
- [ ] **Catherine as executor of will** — Made her the information hub. Too convenient?
- [ ] **John as enforcer/debt collector** — Business arrangement with Stephanie. Is "gangster as business partner" too on-the-nose?
- [ ] **Jack owning the hotel deed** — Stephanie holds his biggest property. Does this work as a motive?
- [ ] **Ross's gambling debts** — Straightforward financial motive. Too obvious as a red herring?

### Alibis
- [ ] **All alibis and their holes** — Each suspect's alibi was designed to have exactly one exploitable gap. Review each in `design.md` for plausibility.
- [ ] **Catherine's 20-minute gap** — When Ross is the killer, Catherine can't account for 20 minutes. Is this too convenient?
- [ ] **Chelsea's unreliable corroboration** — She "assumes" Catherine was there. Solid enough for a mystery?

### Game Mechanics
- [ ] **3-attempt accusation limit** — Too strict? Too generous?
- [ ] **Hints on wrong accusations** — Currently reveals which part (suspect/weapon/location) is wrong. Too much information?
- [ ] **No time limit** — Deliberate for a demo. Would a turn-based deadline add tension?
- [ ] **ACCUSE syntax** — `ACCUSE [suspect] WITH [weapon]` plus player must be in the murder location. Intuitive enough?

## Dialogue (Needs Voice / Personality)

All dialogue was written to be mechanically correct and convey the right information. It needs personality, period voice, and subtext. Currently reads like "competent placeholder."

### Priority: High (defines character)
- [ ] **Ross's emotional range** — His angry/sad/defensive modes all sound similar. Needs the specific cadence of a young 1870s athlete
- [ ] **Viola's deceptive charm** — Her lies should sound elegant and rehearsed. Currently too blunt
- [ ] **John's menace** — Needs to feel genuinely dangerous with fewer words. Currently too explanatory
- [ ] **Jack's bluster** — Should be louder, more specific, more period-authentic
- [ ] **Chelsea's fear** — Needs more physicality (fidgeting, trailing off, pauses)
- [ ] **Catherine's warmth** — She's the most helpful NPC; her dialogue should feel like genuine concern, not exposition delivery

### Priority: Medium (scene polish)
- [ ] **Catherine-Chelsea witnessed scene** — Three lines of dialogue. Needs to feel like a real moment, not a data dump
- [ ] **Between-turn commentary** — Catherine's follow-up about Stephanie's late-night letters. Pacing right?
- [ ] **Viola's greeting** — She initiates when player enters ballroom. Currently too on-the-nose
- [ ] **Influence text** — John's menace aura, Jack's bullying, Ross's intimidation. All need atmosphere

### Priority: Low (informational)
- [ ] **Alibi statements** — Factually correct but could use more personality
- [ ] **Goal pursuit witnessed messages** — "Ross heads to the bar" etc. Functional but flat

## Rooms (Needs Atmosphere)

Room descriptions are architecturally correct but lack period atmosphere. The 1870s reconstruction-era setting should breathe.

- [ ] **Foyer** — Needs the feeling of new money, raw ambition. Gas light, marble, the smell of fresh construction
- [ ] **Bar** — Needs more smoke, conversation noise, the weight of male business culture
- [ ] **Kitchen** — Needs sensory detail (heat, copper, the clatter of prep)
- [ ] **Ballroom** — Needs grandeur and emptiness (it's morning, no event)
- [ ] **Stephanie's room (302)** — Needs to feel like a dead woman's space. Wilting flowers, stopped clock, lingering perfume
- [ ] **Ross's room (304)** — Needs the chaos of a man falling apart
- [ ] **Basement rooms** — Need industrial atmosphere (steam, coal dust, damp)

## Evidence System (Incomplete)

The randomization system exists structurally but several pieces are TODO:

- [ ] **Weapon removal** — When a weapon is the murder weapon, it should be moved to a hidden/evidence location, not just left in place. Currently marked TODO in `randomization.ts`
- [ ] **Location tells** — The murder location should have visible physical evidence (defined in design.md but not yet implemented as entity descriptions)
- [ ] **Killer's alibi modification** — When a suspect is the killer, their alibi corroborator should have modified dialogue. Not yet wired up
- [ ] **Evidence discovery events** — Finding key evidence (locket, letter, theatre program) should trigger conversation unlocks. Not yet implemented

## NPC Behavior Wiring (Incomplete)

The CharacterBuilder definitions are complete, but the runtime wiring has TODOs:

- [ ] **NPC behavior registration** — `onEngineReady()` has TODO for registering NPC behaviors with NpcService. Each NPC needs a behavior implementation that delegates to the character model
- [ ] **Character phase handlers** — TODO for registering propagation, goal pursuit, and influence tick phases
- [ ] **Predicate definitions** — Character constraints like `trusts player`, `threatened`, `cornered` need predicate functions that evaluate actual character state
- [ ] **Trigger activation** — Triggers like `player accuses`, `confronted with program`, `player sympathizes` need to be wired to actual game events
- [ ] **NPC-to-NPC offscreen scenes** — Catherine-Chelsea witnessed scene is defined but not scheduled

## Missing Features

- [ ] **Intro sequence** — Game opening (arrival, morning, manager's request) defined as messages but not wired into the turn cycle
- [ ] **Elevator as a connectable room** — Currently using the staircase landing as the elevator location. A real elevator entity that moves between floors would be better
- [ ] **Score system** — No scoring defined. Should there be points for evidence discovery? Correct deductions?
- [ ] **Save/restore** — Randomized solution needs to survive save/restore cycles
- [ ] **NPC behavior implementations** — The NpcTrait behaviorIds are set but no NpcBehavior implementations exist yet
- [ ] **Confrontation mechanic** — Showing evidence to NPCs (e.g., showing the theatre program to Viola) should trigger state changes. Grammar exists but handler logic doesn't
