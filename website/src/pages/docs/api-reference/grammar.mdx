# Grammar & Commands Reference


This is a reference of all built-in commands available to players. Stories can extend grammar with additional commands.

## Directions

Players can move using direction words or single-letter abbreviations:

| Direction | Abbreviation | Direction | Abbreviation |
|-----------|-------------|-----------|-------------|
| north | n | south | s |
| east | e | west | w |
| northeast | ne | northwest | nw |
| southeast | se | southwest | sw |
| up | u | down | d |
| in | — | out | — |

All directions also work with `go`: `go north`, `go up`, etc.

## Grammar APIs

The platform provides two grammar APIs. The **action-centric** `.forAction()` API (ADR-087) is the primary way to define verb patterns. The **pattern-centric** `.define()` API handles phrasal verbs and complex multi-slot patterns.

### Action-Centric: `.forAction()` (preferred)

Use `.forAction()` when a verb has a simple structure — one or more verb synonyms, an optional entity slot, and optional trait constraints:

```typescript
// Verb aliases with entity slot
grammar
  .forAction('if.action.taking')
  .verbs(['take', 'get', 'grab'])
  .pattern(':item')
  .build();
// Generates: take :item, get :item, grab :item

// No-argument command with aliases
grammar
  .forAction('if.action.inventory')
  .verbs(['inventory', 'inv', 'i'])
  .build();

// Verb + particle + trait constraint
grammar
  .forAction('if.action.switching_on')
  .verbs(['turn', 'switch', 'flip'])
  .pattern('on :device')
  .hasTrait('device', TraitType.SWITCHABLE)
  .build();

// Direction commands
grammar
  .forAction('if.action.going')
  .directions({
    'north': ['north', 'n'],
    'south': ['south', 's'],
    'up': ['up', 'u'],
    'down': ['down', 'd'],
  })
  .build();
```

### Pattern-Centric: `.define()` (for complex patterns)

Use `.define()` for phrasal verbs (multi-word verb phrases), patterns with multiple entity slots, or alternate word orders:

```typescript
// Phrasal verb — can't use forAction because the verb has a space
grammar
  .define('pick up :item')
  .mapsTo('if.action.taking')
  .withPriority(100)
  .build();

// Two entity slots with trait constraint
grammar
  .define('put :item in|into|inside :container')
  .hasTrait('container', TraitType.CONTAINER)
  .mapsTo('if.action.inserting')
  .withPriority(100)
  .build();

// Alternate word order for the same action
grammar
  .define('turn :device on')
  .hasTrait('device', TraitType.SWITCHABLE)
  .mapsTo('if.action.switching_on')
  .build();
```

## Extending Grammar in Stories

Stories add custom commands in `extendParser()`:

```typescript
extendParser(parser: Parser): void {
  const grammar = parser.getStoryGrammar();

  // Action-centric for simple verb patterns
  grammar
    .forAction('dungeo.action.ring_bell')
    .verbs(['ring'])
    .pattern(':target')
    .withPriority(150)
    .build();

  // Pattern-centric for complex or literal commands
  grammar
    .define('say :message')
    .mapsTo('dungeo.action.say')
    .withPriority(150)
    .build();

  grammar
    .define('turn :dial to :number')
    .hasTrait('dial', TraitType.SWITCHABLE)
    .mapsTo('dungeo.action.turn_dial')
    .withPriority(150)
    .build();
}
```

Use priority 150+ for story-specific commands to ensure they take precedence over built-in patterns.

## Parser Notes

- **Multi-object support**: Commands like `take all`, `drop all but lamp`, and `take sword and shield` are handled automatically by the parser.
- **Abbreviations**: Single-letter abbreviations (n, s, e, w, u, d, l, x, i, z, g, q) are recognized.
- **Disambiguation**: When a command is ambiguous ("take ball" with multiple balls present), the parser asks the player to clarify.
- **Articles ignored**: "the", "a", and "an" are stripped — `take the lamp` and `take lamp` are equivalent.
