# Plan — ADR-180 Phase 3d: build.sh cutover

**Date:** 2026-06-18
**Branch:** feat/adr-180-phase3d-cutover
**Predecessors:** 3a CLI bundle (#122), 3b browser (#123), 3c zifmia (#124) — all merged,
all byte-parity gated. **This is the irreversible cutover.**

## Workstreams

1. **`devkit clean`** — build-artifact hygiene: remove `dist/`, `dist-esm/`, `tsconfig.tsbuildinfo`,
   `tsconfig.esm.tsbuildinfo` across packages (the class the 2026-06-17 clean-script fix addressed).
   Delegate to the existing per-package `pnpm -r clean`, plus top-level `dist/`.
2. **`devkit verify`** — `tsf build --npm` + `tsf publish --tag beta --dry-run` (the canonical
   publish-verification, replacing the dead `publish:beta` chain).
3. **Umbrella wiring** — `@sharpee/sharpee` depends on `@sharpee/devkit` (workspace:*) so
   `npm install @sharpee/sharpee` ships the `devkit` bin (ADR Decision 1, deferred from Phase 2).
   devkit is dependency-light (build/test shell out), so no runtime bloat.
4. **GENERATOR flip** — version.ts comment "Auto-generated by build.sh" → "devkit" (the const in
   build.ts). Only valid AFTER build.sh is gone (breaks parity while build.sh still exists).
5. **AC-8 doc sweep** — update LIVING docs to devkit equivalents: CLAUDE.md, package CLAUDE.md,
   docs/guides/build-system.md, docs/guides/project-structure.md, docs/testing/README.md,
   docs/zifmia/install-guide.md, docs/guides/transcript-testing.md.
6. **Delete build.sh** + orphaned scripts — **the irreversible step; explicit sign-off required.**
7. **Final gate** — run all three parity harnesses green ONE LAST TIME immediately before deletion
   (they need build.sh to exist), then delete.

## Decisions needed (sign-off)

- **D1 — fast test bundle** (`build_test_bundle`, `--test` → `dist/cli/{story}-test.js`):
  not an ADR-180 target. CLAUDE.md's transcript testing already uses `node dist/cli/sharpee.js
  --test` (the kept platform bundle). **Recommend: drop it** (delete `test-bundle-template.js`);
  the platform bundle covers transcript testing.
- **D2 — doc sweep scope:** **Recommend: living docs only.** ADRs + `docs/brainstorm/` are
  historical records — they legitimately reference build.sh as it was; rewriting them falsifies
  history. AC-8's intent ("no doc describes a deleted command as the current way") is met by
  fixing instructional docs and leaving the record intact.
- **D3 — orphaned .sharpee scripts** (`validate-bundle-baseline.js`, `generate-meta.js`): support
  the **deferred** `.sharpee` bundle (build_story_bundle). **Recommend: KEEP** (dormant, not dead;
  needed to reconstruct .sharpee later). Only `test-bundle-template.js` is truly dead (per D1).

## Order of operations (safety)

Non-destructive first, deletion last:
1. devkit clean + verify (+ unit tests).
2. Umbrella wiring; `pnpm install`; confirm devkit bin resolves via sharpee.
3. AC-8 doc sweep (living docs).
4. **STOP — confirm with David.**
5. Final parity: run all 3 harnesses green.
6. `git rm build.sh` + `test-bundle-template.js` (per D1); flip GENERATOR const.
7. Re-run devkit build for all 3 targets (now without build.sh) to prove standalone operation.

## Invariants
- build.sh deleted ONLY after the final parity run passes (ADR invariant).
- GENERATOR flip happens only after deletion (else parity breaks).
- No ADR rewritten (history preserved).
