### Step 3 ‚Äî RegionTrait class

**New files**:

#### regionTrait.ts

Follows the exact same structure as `roomTrait.ts`:

```typescript
/**
 * Region trait for geographic groupings of rooms (ADR-149)
 *
 * Entities with this trait represent named spatial regions. Rooms declare
 * membership via RoomTrait.regionId. Regions can be nested via parentRegionId.
 *
 * Public interface: RegionTrait, IRegionData.
 * Owner context: @sharpee/world-model / traits
 */

import { ITrait } from '../trait';
import { TraitType } from '../trait-types';

export interface IRegionData {
  name: string;
  parentRegionId?: string;
  ambientSound?: string;
  ambientSmell?: string;
  defaultDark?: boolean;
}

export class RegionTrait implements ITrait, IRegionData {
  static readonly type = TraitType.REGION;
  readonly type = TraitType.REGION;

  name: string;
  parentRegionId?: string;
  ambientSound?: string;
  ambientSmell?: string;
  defaultDark: boolean;

  constructor(data: IRegionData) {
    this.name = data.name;
    this.parentRegionId = data.parentRegionId;
    this.ambientSound = data.ambientSound;
    this.ambientSmell = data.ambientSmell;
    this.defaultDark = data.defaultDark ?? false;
  }
}
```

`name` is required (no `??` default) ‚Äî a region with no name is a programming error.

#### WorldModel class implementation (after createDoor, before closing brace)

```typescript
/**
 * Creates a region entity with RegionTrait atomically.
 *
 * @param id - Explicit entity ID for the region (e.g., 'reg-underground').
 *             Must be unique; throws if already exists.
 * @param options - Region configuration.
 * @returns The created region entity.
 */
createRegion(id: string, options: RegionOptions): IFEntity {
  if (this.hasEntity(id)) {
    throw new Error(`createRegion: entity '${id}' already exists`);
  }
  const entity = this.createEntityWithId(id, EntityType.REGION, options.name);
  entity.add(new RegionTrait(options));
  return entity;
}
```

Note on `createEntityWithId`: `createEntity()` auto-generates IDs. Region IDs are author-defined strings (e.g., `'reg-underground'`), so WorldModel needs a variant that accepts an explicit ID. Check whether one already exists; if not, add a private helper `createEntityWithId(id, type, displayName)` that skips ID generation and inserts directly into `this.entities`.

Check `WorldModel.ts` for existing explicit-ID creation before implementing ‚Äî the `createDoor` flow uses auto-generated IDs, so this may be new. If a pattern exists (e.g., from save/restore code), reuse it. If not, the helper is ~10 lines.

### Step 7 ‚Äî WorldModel.assignRoom()

**File**: `packages/world-model/src/world/WorldModel.ts`

```typescript
/**
 * Assigns a room to a region by setting RoomTrait.regionId.
 *
 * @param roomId - ID of the room entity.
 * @param regionId - ID of the region entity (must already exist and have RegionTrait).
 * @throws if room does not exist, region does not exist, or region lacks RegionTrait.
 */
assignRoom(roomId: string, regionId: string): void {
  const room = this.getEntity(roomId);
  if (!room) {
    throw new Error(`assignRoom: room '${roomId}' not found`);
  }
  const region = this.getEntity(regionId);
  if (!region) {
    throw new Error(`assignRoom: region '${regionId}' not found`);
  }
  if (!region.has(TraitType.REGION)) {
    throw new Error(`assignRoom: entity '${regionId}' is not a region`);
  }
  const roomTrait = room.get(RoomTrait);
  if (!roomTrait) {
    throw new Error(`assignRoom: entity '${roomId}' does not have RoomTrait`);
  }
  roomTrait.regionId = regionId;
}
```

Also add the import: `import { RegionTrait } from '../traits/region/regionTrait';`

### Step 8 ‚Äî Tests

**New file**: `packages/world-model/tests/unit/traits/region.test.ts`

#### Behavior Statements (required before writing tests per coding discipline)

**RegionTrait constructor**
- DOES: Initializes `name`, `parentRegionId`, `ambientSound`, `ambientSmell`, `defaultDark` from provided data
- WHEN: Called with a valid `IRegionData` object
- BECAUSE: Trait must faithfully store region metadata for later query use
- REJECTS WHEN: (no throws; name is required by type, no runtime guard needed in constructor)

**WorldModel.createRegion()**
- DOES: Creates a new entity with `EntityType.REGION` and attaches a `RegionTrait`; the returned entity has the trait with values matching `options`; the entity is accessible via `world.getEntity(id)`
- WHEN: Called with a unique `id` and valid `RegionOptions`
- BECAUSE: Region entities must be atomically created ‚Äî no half-initialized state (entity without trait)
- REJECTS WHEN: `id` already exists in the world (throws with descriptive error)

**WorldModel.assignRoom()**
- DOES: Sets `RoomTrait.regionId` on the specified room to the given regionId
- WHEN: Both room and region exist and are correctly typed
- BECAUSE: Room‚Üíregion membership is the link that makes all region queries work
- REJECTS WHEN: roomId not found; regionId not found; entity at regionId lacks RegionTrait; entity at roomId lacks RoomTrait

## Known Edge Cases

1. **Explicit IDs**: `createRegion('reg-underground', ...)` uses an author-supplied ID, not auto-generated. Verify whether `WorldModel` has a path for this (check save/restore or entity loading code). If not, the private `createEntityWithId` helper is ~10 lines.

2. **TRAIT_CATEGORIES exhaustiveness**: `TRAIT_CATEGORIES` in `trait-types.ts` is typed as `Record<TraitType, TraitCategory>`. Adding `REGION` to `TraitType` without adding a corresponding entry in `TRAIT_CATEGORIES` will cause a TypeScript error. Step 2 must add both.

3. **RoomTrait rename callers in Dungeo**: The story uses `region:` in `IRoomData` constructor calls. These will produce TS errors after the rename, but they are expected ‚Äî left for Phase 8.

4. **@sharpee/queries stub**: `packages/queries` already defines `ENTITY_TYPE_REGION = 'region'`. After Step 1, the value in `EntityType.REGION` will match this constant ‚Äî no queries change needed in Phase 1, but confirm no type collision.
