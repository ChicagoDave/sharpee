# Session Plan: Chord Door Loading (ADR-234 implementation)

**Created**: 2026-07-18
**Overall scope**: Give Chord (`.story` author language) the ability to load doors ‚Äî closing the last ‚ùå of the parity audit's action table and the door line of ADR-233's G1 parity gate. One authoring form ships: the exit-line sugar (`north to the Hall through the oak door`, reverse side inferred), per David's 2026-07-17 ruling that struck the `between`-placement form from scope. The platform substrate (`DoorTrait`/`DoorBehavior`, `createDoor`, `connectRooms`'s bidirectional convention, going.ts's dead `door_closed`/`door_locked` branches) is already fully built ‚Äî this plan is a compiler-frontend (`packages/chord`) + loader (`packages/story-loader`) surface, plus the ADR-237 boundary work (helpers unravel + `connectRooms` door growth ‚Äî Phase 1, REVISED 2026-07-18) that gives door wiring one first-class home and removes the loader's `@sharpee/helpers` dependency. **Bounded contexts touched**: `packages/world-model` (`connectRooms` grows an optional `doorId` ‚Äî ADR-237 D4; its acceptance is the platform-change discussion), `packages/helpers` (`DoorBuilder` delegates its wiring tail to `connectRooms`), `packages/chord` (lexer/parser/AST/analyzer/IR for the `through` exit-line tail and the R3 config-value grammar), `packages/story-loader` (helpers unravel + `buildEntity` on direct trait composition per ADR-237 D2/D3; door entity construction, kind-scoped lockable default, R3 field routing), `docs/reference/chord-grammar.md` + `chord.ebnf` (grammar docs), `docs/architecture/chord-grammar-changes.md` (ratchet log, R2/R3).
**Key domain language**: door (kind noun, already in `KIND_NOUNS`), `through` exit-line tail (the one door-relationship form), via (the platform's exit‚Üídoor pointer, `RoomBehavior.setExit`'s third argument), `connectRooms(‚Ä¶, doorId?)` (the one exit-wiring implementation, plain or doored ‚Äî ADR-237 D4), kind-scoped trait default (a door's `lockable` starts locked, diverging from the trait-wide default), ratchet entry (R2 `through`, R3 keyless config).

- **Budget**: 350
- **Domain focus**: `packages/chord` (lexer `through` keyword, `ExitDecl` AST gains a `via: NameRef | null`, IR carries the unresolved via-reference, analyzer never-guess gates) and `packages/story-loader` (door entity construction ‚Äî `SceneryTrait` + `OpenableTrait` starting closed, per D4's baseline ‚Äî and door wiring via Phase 1's `wireDoor` seam once both the door and both rooms are resolved).

  - IR: `via` carried as an unresolved name reference at IR stage (same forward-ref shape used elsewhere ‚Äî e.g. region `containing` members) resolved at load time.
  - Loader: the `case 'door':` LoadError (`loader.ts:943-944`) is replaced.
