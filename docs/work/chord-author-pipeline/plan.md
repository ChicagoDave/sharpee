### Phase 2: Chord-first scaffold + build path (the core gap)
- **Tier**: Large
- **Budget**: 350
- **Domain focus**: `packages/devkit` (`standalone/init.ts` + a new Chord template, `standalone/build.ts`, `standalone/build-browser.ts`) — reuses `packages/chord`'s `compile()`, `packages/story-loader`'s `createStory()`, and `packages/bootstrap`'s `assembleGame()` exactly as `scripts/bundle-entry.js` and `sharpee compose` already do; no changes to those three packages are expected, this is wiring devkit to plumbing it already depends on.
- **Entry state**: Phase 1 done (or independent — this phase does not depend on `test`/`play` or the doc sweep, but should follow Phase 1 in the same session bundle since both touch `cli.ts`).
- **STOP-AND-ASKS RESOLVED (David, 2026-07-18, session 80ff54)**:
  1. **Scaffold default — RULED: Chord is the default.** `sharpee init` scaffolds a `.story` project; `--ts` keeps the TypeScript scaffold as the explicit opt-out.
  2. **Compile-time boundary — RULED (against the plan's recommendation): the compiler ships client-side, and the bundle ships the `.story` SOURCE.** `sharpee build --browser` produces `dist/web/` carrying the author's `.story` text plus a client bundle containing `@sharpee/chord` + `@sharpee/story-loader` + `@sharpee/bootstrap` + engine; the CLIENT compiles at boot (`compile(story.story) → IR → createStory → assembleGame`). Chord is browser-safe by design (its package invariant), so this is a supported shape. Implementation note (not a ruling change): `sharpee build`/`build-browser` still RUN the compiler at build time as a fail-fast validation gate — diagnostics surface on the author's machine, never first as a broken page — but the shipped artifact is the source, not the IR.
- **Deliverable**: A new Chord story template (starter `.story` file + minimal `package.json` scaffold, parallel to the existing `templates/story/` TS templates) plus the Chord-default scaffold wiring per ruling 1; `sharpee build` and `sharpee build --browser` detect a `.story` source (vs. `src/index.ts`) and, when found: run the compiler as the fail-fast validation gate, then (per ruling 2) emit `dist/web/` carrying the `.story` source + a self-contained client bundle (chord compiler + story-loader + bootstrap + engine + browser client) that compiles at boot. The existing TS scaffold/build path stays fully intact and unbroken (regression, not replacement).
- **Exit state**: A freshly `sharpee init`-scaffolded Chord project builds via `sharpee build --browser` into a `dist/web/` bundle that boots to a playable browser story with zero TypeScript compilation involved in the story's own logic. REAL-PATH test: an actual fixture `.story` project run through the real `sharpee build`/`build-browser` commands (not a stub of `chord.compile`/`createStory`/`assembleGame` — Integration Reality, rule 13a), plus a regression run confirming the pre-existing TS scaffold/build path is unaffected.
