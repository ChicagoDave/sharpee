# Plan: Revert ADR-143 Direction Vocabularies

## Context

ADR-143 (Location-Relative Direction Vocabularies) was partially implemented to support naval/minimal/custom direction word sets per region. The ADR review scored 8/15 — the implementation violates language layer separation (English strings in world-model), most of the designed functionality (region assignment, boundary handling, automatic vocab switching on move) was never built, and no stories use it. The working portion is essentially a manual global switch that no story calls.

Ripping this out restores a clean, static compass direction system and removes a broken architectural precedent. The ADR can be re-implemented properly if/when a story actually needs it.

## Scope

**8 packages touched, 2 test files deleted, 2 doc files updated/removed.**

No stories reference `world.directions()` or `useVocabulary()` — zero consumer-side changes.
