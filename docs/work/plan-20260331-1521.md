# Fix: Capability Dispatch Message Key Double-Prefix Bug

## Context

Capability dispatch actions (lowering, raising) produce blank output because message keys are double-prefixed. `loadActionMessages()` stores `${actionId}.${key}`, but lowering/raising define fully-qualified keys like `'if.lower.lowered'`, producing `if.action.lowering.if.lower.lowered` ÔÇö which nothing ever looks up.

Bug writeup: `docs/work/bug-capability-dispatch-message-keys.md`
Branch: `fix/capability-dispatch-message-keys`

## Approach

Align capability dispatch with the convention used by all standard actions: **short message keys in language files, prefixed with actionId at emit time**.

Modern standard actions (taking, opening, closing) use a conditional pattern:
```typescript
const messageId = error.includes('.') ? error : `${context.action.id}.${error}`;
```
This allows story code to provide fully-qualified keys (bypassing the prefix) while short keys get auto-prefixed.

Apply this same pattern inside `effectsToEvents()` in capability-dispatch.ts so ALL behavior effects (report + blocked) get short keys auto-prefixed. Story-specific keys with dots (like `'dungeo.basket.player_transported'`) pass through untouched.

### 3. `packages/stdlib/src/actions/capability-dispatch.ts`
Two changes:

**a) `effectsToEvents()`** ÔÇö auto-prefix short messageIds:
```typescript
function effectsToEvents(
  effects: CapabilityEffect[],
  context: ActionContext,
  actionId: string
): ISemanticEvent[] {
  return effects.map(effect => {
    const payload = { ...effect.payload };
    if (payload.messageId && !payload.messageId.includes('.')) {
      payload.messageId = `${actionId}.${payload.messageId}`;
    }
    return context.event(effect.type, payload);
  });
}
```

Update all 3 call sites (report, blocked-with-behavior, blocked-default) to pass `config.actionId`.

**b) Default blocked path** ÔÇö use short key for messageId:
```typescript
messageId: result.error || config.cantDoThatError,
// Already short after step 4, and effectsToEvents prefixes it
```
