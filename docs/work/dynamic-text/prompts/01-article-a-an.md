# Win 1 / 5 — a / an, chosen for you

The dynamic case: the author writes **one** line; the indefinite article flips between
"a" and "an" by the item's sound — for items they never named at write time.

```text
Create a clean, minimal editorial-infographic image, single idea. Flat vector style,
light background (#F7F8FA), one accent indigo (#4B47B8), neutral grays. Lots of
whitespace. Sans-serif for prose, monospace for the template line. Aspect ratio 16:9.
Small badge top-right: "Win 1 / 5 · a / an".

Top, centered: ONE monospace template card, the placeholder {a item} highlighted indigo:
   You found {a item}!

A thin arrow fans DOWN to three small output cards in a row, each a finished sentence,
with the chosen article highlighted in indigo:
   "You found a hat!"
   "You found an umbrella!"
   "You found a sword!"

Title across the top: "One line. The article corrects itself."
One gray caption bottom-center: "a vs an is chosen from the item's sound at render
time — so the same authored line is right for every object, including ones added later."

Keep it to: one template card → three output cards. No tree, no engine boxes.
```

Notes: the win is "you can't hand-write this once for unknown items." Static prose
would just be typed; this only matters because `item` is dynamic.
