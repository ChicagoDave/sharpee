# Session Plan: Implement ADR-227 (Death Triggers Use the Prescribed Logic Surfaces)

**Created**: 2026-07-16
**Overall scope**: Re-home the ~15 Dungeo death sites (currently bespoke `killPlayer` re-points from the ADR-224 Phase-4 migration) onto their ADR-090-correct prescribed logic surface; build the three Chord authoring constructs (`kill the player`, `<direction> is deadly:`, `deadly:`) that lower to those same surfaces with no new platform mechanism; and fix the one real platform gap (`basicNpcResolver` still emitting legacy `if.event.death` instead of `killPlayer`). No new primitive is introduced anywhere — `killPlayer` (ADR-224) and `DeadlyRoomTrait` (ADR-224 Phase 2) are already built; every phase below is either a re-home onto an *existing* surface or new Chord grammar lowering to one.
**Bounded contexts touched**: `stories/dungeo` (falls, grue, cake, melt-glacier re-homes — story-level), `packages/extensions/basic-combat` (`basicNpcResolver` fix — platform), `packages/chord` (`parser.ts`/`catalog.ts`/`ast.ts`/`ir.ts` — new grammar, platform), `packages/story-loader` (`runtime.ts`/`loader.ts` — new lowering, platform). No `packages/world-model` changes are authorized by this ADR (killPlayer + DeadlyRoomTrait already exist). ~~No stdlib changes~~ **Amended 2026-07-16: two `packages/stdlib` seam-fixes approved by David for Phase 2** — `eating.ts` ADR-118 hook wiring and `throwing.ts` thrown-item interceptor resolution (both close silently-dead Chord `on <action> it` seams; see Phase 2).
**Key domain language**: `killPlayer` (the single mortality sink, ADR-224), `DeadlyRoomTrait` (verb-allowlist no-escape room), deadly *exit* (a lethal direction, new concept this ADR names but does not newly implement as a mechanism — a ParsedCommandTransformer redirect to the generic `DEADLY_ROOM_DEATH_ACTION_ID`; amended 2026-07-16, originally misplanned as a `going`-interceptor), Capability Dispatch (ADR-090 — entity owns a verb's lethal meaning), Action Interceptor (ADR-118/208 — block/redirect a standard action *resolving on a real entity*), the ADR-090 decision tree (the rule that picks a site's surface).
