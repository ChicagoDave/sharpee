Architecture Layers

Layer 1 Client Applications

User-facing applications that interact with the Sharpee platform. Each client provides a different interface for playing interactive fiction games.

  • Web Client: React-based browser interface
  • CLI Client: Terminal REPL for command-line play
  • Electron: Native desktop application
  • VS Code Extension: IDE integration for authors

Layer 2 Stories / Games

Individual interactive fiction stories created using the Forge API. This layer represents the actual games that players experience.

  • Use Forge's fluent API for world building
  • Can load extensions for additional features
  • Define rooms, objects, and interactions
  • Configure game-specific behavior

Layer 3 Forge (Author API)

High-level API for game authors. Provides intuitive, fluent interfaces for creating interactive fiction without dealing with low-level details.

  • ForgeEntity: Fluent API for creating objects
  • ForgeWorld: World builder and extension loader
  • Templates: Pre-built objects and rooms
  • Story Helpers: Scene and dialogue builders

Layer 4 Standard Library

Core IF implementation layer. Handles all the standard interactive fiction mechanics, parsing, and game logic.

  • Actions: All standard IF commands
  • Services: Parser, scope, turn management
  • Text System: Event-to-text rendering
  • Bidirectional connection with Language/Parser

Layer 5 World Model Enhanced

Defines the structure of the IF world. Contains traits, behaviors, and the new extension system.

  • Core Traits: Only fundamental IF concepts
  • Static Behaviors: All logic uses static methods
  • Extension System: Infrastructure for plugins
  • Entity Management: IFEntity and EntityStore

Layer 6 Core (IF-Agnostic)

Foundation data store layer. Provides basic entity management and event system without any IF-specific concepts.

  • Entity System: Base entity and world storage
  • Event System: Pub/sub infrastructure
  • Rule System: Condition evaluation engine
  • No IF-specific logic at this level

Extension System Architecture

The extension system allows third-party developers to add features without modifying core. Extensions can:

  • Define new traits (e.g., DaemonTrait, DialogueTrait)
  • Add new actions and commands
  • Register event handlers
  • Provide services that integrate with game loop
  • Include their own language/text templates

All extension components are namespaced (e.g., sharpee.daemon.RANDOM_MESSAGE) to prevent conflicts.
