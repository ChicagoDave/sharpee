## Key Decisions

### 1. Plain HTML + CSS over Astro
The site is 6 pages. The previous Astro site added build complexity without meaningful benefit at this scale. Plain HTML loads instantly and is trivially deployable to GitHub Pages.

### 2. components.js for Shared Layout
Sidebar and footer are injected at page load from a single JS file rather than duplicated in every HTML file. Keeps the 6-page site maintainable without a build step.

### 3. 17 Separate Pages for Zoo Tutorial (not single-page tabs)
Individual pages are easier to link to directly, easier to maintain, and avoid large JS tab logic. Numbered step tabs in the left column provide the same visual affordance as a single-page wizard.

### 4. Different Accent Color per Theme
Champagne gold (#f2d280) in dark mode and sky blue (#3b82c4) in light mode — chosen because gold reads poorly on white backgrounds and blue reads poorly on near-black backgrounds. Each theme gets a purpose-built accent.

### 5. Author-Facing Framing (not Developer-Facing)
The site is for writers who want to make IF, not for developers contributing to the platform. Getting Started, tutorials, and community are the priority pages; API reference is linked from the sidebar as a secondary resource.
