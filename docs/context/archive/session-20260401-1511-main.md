## Key Decisions

### 1. bridge and runtime are not npm packages
`publishConfig` was present in both `packages/bridge/package.json` and `packages/runtime/package.json`, which would have caused tsf to publish them to npm. Both packages are internal infrastructure and should not be distributed. Removed `publishConfig` from both.

### 2. tsf replaces publish:beta and npm-latest.sh
The project has migrated to tsf for npm publishing. The old `publish:beta` npm script and `scripts/npm-latest.sh` are dead code. Do not add `publishConfig` entries without verifying intent with tsf's package selection logic.

### 3. Declaration merging for EventDataRegistry (zero as-any)
The audio event registry extension uses TypeScript `declare module` augmentation to extend `@sharpee/core`'s `EventDataRegistry`. This approach requires no `as any` casts and integrates cleanly with the existing typed event system.

### 4. runtime build errors are pre-existing ‚Äî do not attempt to fix
The `Story` type resolving to `never` in `packages/runtime` is a known issue. Investigation was inconclusive and the user confirmed it should not be touched in this session.
