## Key Decisions

### 1. sharpee-play:// custom scheme instead of file://
`file://` gives WKWebView a null origin, which causes localStorage to throw a SecurityError before the engine loads. A registered WKURLSchemeHandler serving from the bundle path fixes the origin and requires zero changes to the built web client.

### 2. JS hook sends message + stack, not stack alone
WebKit's `Error.stack` omits the message line. Sending only stack caused the capture to arrive as "Play runtime error" with no translatable content. Hook now sends both fields separately.

### 3. Translation layer is the value; view is dumb
The reframe from "error list" to "translation layer + browsable list + diagnosis panel" separated substance from display. SharpeeErrorTranslator owns the Sharpee vocabulary mapping; the views are read-only projections.

### 4. thealderman audit approach: fix the concrete hallucination; file issue for drift
`world.getLastCreatedEntityId()` was a clear hallucination with a known correct pattern. Fixed inline. The `.becomes()` on ResponseChainBuilder was API drift requiring a broader audit → filed as GitHub issue #130 rather than guessed-at inline.
