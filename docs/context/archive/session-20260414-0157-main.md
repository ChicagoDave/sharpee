## Key Decisions

### 1. World Explorer Tree View to Be Replaced in Tier 3
The tree view implemented in Phase 3 does not scale to large stories like Dungeo (100+ rooms, hundreds of entities). Tier 3 will replace it with a webview-based World Index panel. Entity Autocomplete (Phase 4) is retained as-is since it scales independently of display surface.

### 2. Shared Cache Between World Explorer and Autocomplete
Rather than each provider invoking the CLI independently, `WorldExplorerProvider.getWorldData()` serves as the single cache point. `SharpeeCompletionProvider` calls into it. This avoids redundant spawns and keeps world data consistent within a VS Code session.

### 3. Platform Changes Identified for Tier 3
Tier 3 reference panels will require richer introspection output from `--world-json` / a new `--introspect` flag. Needed: trait definitions and properties, registered capability behaviors, grammar patterns and action group membership, language registry with per-message-ID resolved text, and region metadata for room grouping. These are platform changes requiring discussion before implementation.
