## Key Decisions

### 1. Dedicated `@sharpee/bridge` Package
A separate package avoids polluting the browser-targeted `@sharpee/runtime` with native-app concerns. The bridge has a different build target (Node/Electron subprocess vs browser IIFE) and different consumers.

### 2. Platform Events Cross the Bridge
Rather than inventing new outbound message types for save/restore acknowledgements, platform events (`platform.save_requested`, `platform.restore_requested`, etc.) travel over the bridge alongside domain events. This keeps the event model uniform and avoids a parallel message taxonomy.

### 3. No Session Multiplexing
Dropping Q4 simplifies the protocol considerably — no session IDs, no routing, no concurrent-game bookkeeping. The host process model handles isolation at the OS level.
