## Completed

### Root Cause Identified
The early string-collapse bug in `lang-en-us/formatters/registry.ts` (`applyFormatters`, signature `(string | number | EntityInfo | EntityInfo[]) => string`) was named as the single root cause behind all dynamic-text gaps: once `{the:item}` collapses to `"the cabinet"`, article agreement over adjectives, counting/grouping, pronoun reference, and multi-contributor composition are all impossible downstream. This framing made five of the six proposed A–F ADRs redundant — they were each patching a symptom of the same cause.

### Design Pivot: Phrase Algebra
The six-ADR bolt-on plan (`dynamic-text-proposed-solutions.md`) was superseded by a unifying phrase algebra: text is a tree of composable **phrases** (each an immutable value that defers realization and carries grammatical metadata) realized at end of turn by **one per-locale Assembler**. ADR-190's `EntityInfo` was identified as "the first phrase" — the new model completes what was already started. Authored to `docs/work/dynamic-text/phrase-algebra-design.md`.

### Decisions D1–D5 (recorded in design doc)
- **D1**: Authoring surface is a string that parses to a phrase tree (not pure programmatic).
- **D2**: Placeholder grammar: `NounPhrase` is the unprefixed default (`{item}`, `{the item}`, `{capitalize the item}`); all other kinds use explicit full-word prefixes (`{pronoun:it}`, `{number:coins words}`, `{contents:box}`, `{slot:detail}`, `{verbatim:banner}`). No `:` chains, `?`, `|`, or `#` — clean break from the old parser.

## Key Decisions

### 1. Phrase Algebra Over Six Bolt-On ADRs
Rather than ADRs A–F each adding a mini-language around the `:` split parser, a single model handles all cases. The six ADRs become named atoms (Literal, NounPhrase, List, etc.) and combinators (Sequence, Contents, Slot) of one closed union. This keeps logic in code (Optional/Choice are named producers, not in-string control flow) and keeps templates near-dumb — consistent with Sharpee's standing philosophy.

### 2. v2 Branch Strategy
The phrase algebra is a breaking change to the formatter contract. Rather than trying to migrate incrementally on main, a long-lived `v2` branch is the right home. The book (em-dash editorial chapters) stays on 1.x with periodic forward-merge; forward work originates in v2. Nothing is cut until the first breaking commit.

### 3. Phrase Contracts in `@sharpee/if-domain`
The natural home for language-neutral phrase types is `@sharpee/if-domain`, which already owns entity and trait contracts. A new `@sharpee/text` package was rejected because it collides with the existing `text-service`/`text-blocks` family and would add an unnecessary dependency hop.
