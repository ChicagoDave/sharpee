## Key Decisions

### 1. vite-node CJS workaround scoped to test runner only

The `createRequire(__filename)` path for engine-runtime classes in `turn-executor.ts` is explicitly marked as a test-runner-only workaround. Production bundles go through esbuild which resolves the double-export cleanly. Documented inline so a future reader doesn't mistake it for a design pattern.

### 2. `esmExtensions` as a separate field in tsf, not an `imports` enum value

The existing `imports` field governs cross-package specifier strategy; `esmExtensions` governs intra-package relative path extension-appending. They are orthogonal concerns: a consumer can use `imports: "preserve"` with `esmExtensions: true`. Merging them would have required every existing config to be reviewed. See tsf session summary for full rationale.

### 3. `SceneryTrait` removed from tiny-story marker stone

Mutation-verification showed the original rehydration test was tautological — a `SceneryTrait` item cannot be taken, so "player doesn't hold marker" is the initial state and proves nothing about restoration. Removing the trait and asserting the player holds the marker after turn-2 `take` provides a genuinely falsifiable restore assertion.

### 4. tsx → tsc + node for perf harness

tsx's aggressive module-path caching served stale `dist-esm/` shapes after `tsf build` regenerated them. `tsconfig.perf.json` + `tsc && node` bypasses the cache. This is a diagnostic footgun to remember whenever `@sharpee/*` ESM builds are regenerated.
