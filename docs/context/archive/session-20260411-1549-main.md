## Key Decisions

### 1. Annotate Rather Than Eliminate Some Casts

Two benchmark casts and the prototype augmentation cast are structurally necessary.
Rather than contorting the architecture to remove them, they are annotated with
`// justified: <reason>` comments and left in place. ESLint `no-explicit-any: error`
will still catch any new unjustified casts.

### 2. Delete `parser-demo.ts` Rather Than Repair It

The file imported types that no longer exist and could not compile. Fixing it would
require a non-trivial refactor of example code that is not tested and not imported
anywhere. Deleting it removes 5 casts cleanly and avoids maintaining a broken file.

### 3. Fix `cli-platform.ts` via Field Name Correction, Not Type Widening

The four casts all accessed a field named `.data` that does not exist on the platform
operation type. The correct field is `.payload`. This was a latent bug hidden by `as any`
— removing the cast revealed and fixed the bug at the same time.

### 4. `ResponseStateMutation` Type Tightening Surfaces `thealderman` Latent Bugs

Changing `mood: string` to `mood: Mood` in `dialogue-extension.ts` exposes that the
`thealderman` story passes mood strings not present in the `Mood` enum. These are real
type errors that would cause runtime mismatches. Left as known follow-up items.

### 5. Rewrite TestPlatform With Public Engine API

The TestPlatform subscribed to engine internals via `(engine as any).events`. The engine
exposes a public `on(eventName, handler)` method that achieves the same result. Switching
to it makes the test platform resilient to future engine internals reorganization.
