## Key Decisions

### 1. Post framing pivot: author-enabled UX over paradigm shift

The draft framed the change as a technical paradigm shift ("TextBlocks to Channels"). David retitled and reframed it around author-enabled client architecture — the angle that highlights the story author's ability to customize UI surface and gestures. This is the correct long-term framing for the blog and aligns with existing project memory. Future post drafts should lead with author control, not wire-protocol mechanics.

### 2. Breadcrumb-driven mmdc workflow recovery

Rather than re-discovering the Mermaid rendering setup from scratch, the `docs/_archived/puppeteer-config.json` file from a February 2026 session served as a breadcrumb. The same npx-installed puppeteer carried over to the Alderman JSX rendering. This confirms the value of leaving archived config artifacts in place even after a workflow changes.

### 3. Self-contained HTML wrapper for JSX component preview

Rather than wiring up a build toolchain to render the detective-sheet JSX, a self-contained HTML wrapper (React 18 + Babel-standalone via CDN, JSX inlined verbatim) was used for a one-shot puppeteer capture. This is appropriate for documentation-only previews where the component is not yet integrated into a build pipeline. The approach is intentionally disposable — no build artifact, no registry entry.

### 4. Diagram assets colocated with walkthrough docs

All `.mmd` sources, `.png` renders, and `.html` wrappers live in `docs/work/channel-io-unification/diagrams/` alongside the walkthrough markdown files. This keeps the visual assets traceable to their source diagrams and avoids scattering them into a separate `docs/images/` tree.
