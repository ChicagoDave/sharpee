## Key Decisions

### 1. Capability Gating Stays Off the Wire
Gating metadata lives in a separate internal map keyed by channel id, populated by `registerChannel`'s optional `gatedBy` argument. The wire `ChannelDefinition` has no producer-only fields. This keeps the wire contract clean and means clients never receive data that is only meaningful to the producer.

### 2. Module-Level Singleton With `resetSession()`
Phase 1 uses a module-level singleton rather than a factory. `resetSession()` allows AC-12 replay and ADR-164 stateless-worker-per-request reuse by clearing all mutable state between sessions. A factory pattern can replace this in a later phase if concurrency demands it; for now the singleton matches ADR-163 §12's imperative API exactly.

### 3. Single-Call-Per-Session for `produceCmgtManifest`
The manifest is frozen on the first successful call. A second call in the same session throws. This enforces §11's "frozen for the session" invariant. Multi-user (ADR-164) achieves per-client manifests by calling `resetSession()` per worker, not by making `produceCmgtManifest` callable multiple times.

### 4. Decoder is Unrecoverable on Error
Once the decoder enters the `'error'` state, all subsequent packets are silently ignored. The caller must drop the decoder instance and create a fresh one. This simplifies downstream error handling — no partial-recovery edge cases.

### 5. Turn Producer Accepts `events` and `world` Parameters (Routing Deferred)
`produceTurnPacket` accepts `events` and `world` but does not yet route them. Phase 1 routing is text-block-only. The parameters are present so Phase 2 media-channel routing can be added without changing the call signature.

### 6. Score Channel Uses Custom Extractor
The stdlib emits score as `'42'` or `'42 / 100'`. Rather than changing the stdlib format, the platform rule for `status.score` includes a custom extractor that parses both forms. This avoids a cross-package change that would widen the scope of Phase 1.
