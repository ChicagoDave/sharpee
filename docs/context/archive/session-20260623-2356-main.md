## Key Decisions

### 1. Built-in themes ship with platform-browser, not as packages
ADR-188 delivery model revised 2026-06-23: the `@sharpee/theme-*` package mechanism is retained only for future third-party themes. Built-ins travel with the engine package, selected by id. This eliminates install/publish friction for standard themes.

### 2. zoo-sunny kept rich
User chose to keep `familyzoo`'s `zoo-sunny` theme with its full green-chrome CSS flourishes rather than reducing it to a token-only block.

### 3. Author-theme vs built-in contract
Authors apply built-ins by id string; their own themes are inline `{id,name}` objects in `sharpee.themes`. The distinction drives different build-time behavior: built-in ids → CSS copy + link from platform-browser; inline objects → menu entry only (CSS lives in the author's own override file).

## Follow-up (post-finalize, post-publish)

After the user published `platform-browser@1.1.2` + `devkit@1.1.3`, the Family Zoo
build (the first real consumer of the *published* packages) surfaced a **publish-path
bug** the in-repo tests masked:

- **Root cause:** tsf flattens each package's `exports` to `.`-only on publish (it ships
  a flat, single-entry package with assets as plain files — by design). devkit's
  `resolveEngineStylesDir` located platform-browser's `styles/` via
  `require.resolve('@sharpee/platform-browser/package.json')` — a subpath that doesn't
  survive publish → `ERR_PACKAGE_PATH_NOT_EXPORTED`. The in-repo tests only ever resolved
  the *source* package (which still has the subpath), so it slipped through.
- **Fix (devkit):** resolve the package's `.` entry (the one export tsf preserves) and
  walk up to the dir holding `styles/engine.css` — works for both the published flat
  layout and the in-repo `dist/` layout. No tsf change, no platform-browser rev.
- **Regression guard:** new `resolve-engine-styles.test.ts` (3 tests) exercises flat +
  nested + not-found with synthetic fixtures — covering the exact gap.
- Bumped + published `devkit@1.1.4`; Family Zoo `devkit` dep → `^1.1.4`.
- **Family Zoo build verified GREEN against published packages:** `themes/` has the 4
  built-in CSS + namespaced system-6 fonts; zoo-sunny is menu-only (CSS in
  `familyzoo.css`, linked last); menu = Classic + 5; cascade order correct.
- **README sweep:** added the *Theme engine + built-in themes (ADR-188)* capability to
