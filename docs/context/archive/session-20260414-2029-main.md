## Key Decisions

### 1. workspace:* Over file: Protocol
Switched all `file:../../../packages/...` dependency references to `workspace:*`. The `file:` protocol breaks when moving packages or building from different working directories; `workspace:*` is the correct pnpm monorepo pattern and resolves correctly regardless of physical path.

### 2. resolve_story_dir() for Build Generalization
Rather than duplicating browser build logic for tutorials, extended the existing `resolve_story_dir()` helper in `build.sh`. This keeps one code path for all story types and avoids the pattern of hardcoded `stories/` assumptions spreading through the build system.

### 3. Tutorial Stages as Independent Projects (Deferred)
Recognized that build-flag approaches or a single monorepo with conditional compilation are wrong for tutorial audiences. The right solution is standalone, self-contained projects per stage. This decision is deferred to John to design the stage breakdown and learning arc.
