## Key Decisions

### 1. Trait Fields over Event Handlers for Computed Descriptions
The prior approach used an event handler (`openable-description-handler.ts`) that intercepted examining events and swapped description text. The new approach puts the description variants directly on the trait and computes them at the getter level. This eliminates a class of event handler that existed solely to compensate for a gap in the data model.

### 2. $teleport Removal Revealed Real Bugs
The $teleport shortcut mechanism had been masking two genuine world-model defects. Removing the shortcuts and writing proper navigation sequences forced the transcripts to exercise real pathfinding, which surfaced the missing exit and carousel logic bug. The bugs were fixed in the same PR rather than deferred.

### 3. Documentation as Code Review
All six issues resolved across recent sessions (ISSUE-063, 064, 065, 068, 070, 054) were originally surfaced by writing the CS Foundations document. Explaining how code works in precise prose forces careful reading that catches architectural drift and implementation gaps. This is now an explicit technique in the project methodology.
