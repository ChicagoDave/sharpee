## Key Decisions

### 1. Code is Source of Truth
ADRs document rationale and historical intent; the actual TypeScript implementation wins on any divergence. The spec flags known divergences rather than silently reconciling them.

### 2. Contract-Level Granularity
Every trait and action is named and its contract described, but not every individual field is enumerated. The goal is a re-implementation guide, not a TypeScript-to-English transcription.

### 3. 49 Standard Actions, Not 43
CLAUDE.md states 43 stdlib actions (the refactoring tally). The actual count when meta-commands are included is 49. The spec uses the higher number and notes the discrepancy.

### 4. Code-vs-ADR Divergences Flagged
Two confirmed divergences documented:
- `WorldSerializer` emits entities as an array, not a record (ADR-033 describes a record shape)
- Spatial index shape at runtime is `{parentToChildren, childToParent}`, not the declared `{entities, locations, relationships}` triple
