## Completed

### Part 1: Capability Dispatch Bug Fix (commits 2f5e893c, f6e2ce9b, 9a6ebff7)

Fixed blank output from lowering/raising capability dispatch actions. Root cause was double-prefixed message keys: `loadActionMessages()` in the language provider prepends `${actionId}.` to every key, but `lang-en-us` lowering and raising files used fully-qualified keys like `'if.lower.lowered'` rather than short keys like `'lowered'`. The stored key became `if.action.lowering.if.lower.lowered` — a path nothing ever looked up.

Fix applied across 6 files: both lang-en-us action files (short key form), `capability-dispatch.ts` (updated `effectsToEvents()` to auto-prefix keys without dots), both stdlib lowering/raising action files, and the dungeo basket-elevator behaviors. Verified with basket-elevator.transcript (9 pass). Rebuilt and deployed to website.

### Part 2: Entropy Design Revision (commits 6df3896e, 2a8dca7c — prior session)

Revised `stories/entropy/design/design.md` based on David's new direction.

**Major arc change**: Cut the neutral planet as a playable region entirely. Replaced the old Act 4 (physical travel to neutral planet, social infiltration, disguise mechanic) with a 2-turn AE Field temporal projection mechanic.

## Key Decisions

### 1. Closed Time Loop as Act 4 Replacement
The neutral planet arc required a playable region (rooms, NPCs, social puzzle) that added scope without adding narrative punch. The UPDATE MEMORY mechanic achieves the same information goal — learning the device's hiding place — with a 2-turn mechanic that is thematically resonant and mechanically elegant. Past-self chose the hiding spot; only Chrysilya can know where that is.

### 2. Short Keys in Capability Dispatch Language Files
All stdlib actions use short keys in language files; `loadActionMessages()` handles the prefix. The capability dispatch lowering/raising files were inconsistent. The fix aligns them with the convention rather than changing the convention. Story-specific fully-qualified keys (containing dots) continue to pass through unchanged via the conditional in `effectsToEvents()`.
