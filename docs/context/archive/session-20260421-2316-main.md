## Key Decisions

### 1. Port is 8080, not 3000
The plan's prose said default port 3000. The actual `config.ts` defaults the port to **8080** and the `docker-compose.yml` maps `8080:8080`. All four documents use 8080 consistently. This would have been a broken install guide if taken from the plan prose without source verification.

### 2. CAPTCHA provider enum
The plan said `friendly_captcha`. The actual config loader accepts `turnstile | hcaptcha | friendly | none`. Documented the correct enum values. Also documented that `CAPTCHA_BYPASS=1` is an env var mechanism (distinct from setting `captcha.provider: none`) — needed for the smoke-test script to work, and labeled "never in production" in the guide.

### 3. No published image — clone-and-build only
The plan described `docker compose pull` as the upgrade path, implying a registry image. No such image is published at v0.1. The install guide documents clone-and-build as the only supported path. The upgrade guide documents `git pull` + rebuild rather than `docker compose pull`.

### 4. Build context is repo root
The `Dockerfile` uses `../..` as build context, which means operators must clone the full monorepo — they cannot download just `tools/server/`. The install guide explicitly calls this out.

### 5. Preferred backup method: sqlite3 hot backup
Chose `sqlite3 .backup` as Method 1 because it is hot (no downtime), WAL-aware, and produces a consistent snapshot. Tar-of-volume is Method 2 because it requires stopping the container to avoid WAL inconsistency, but some operators may not have `sqlite3` CLI available inside the container.

### 6. CAPTCHA_BYPASS warning placement
The `CAPTCHA_BYPASS=1` path is documented in the install guide's smoke-test section with an explicit "never in production" callout. It is needed because the smoke test script requires bypassing CAPTCHA to create a test room, and operators will hit this path when verifying their install.

### 7. Explicit warning against docker compose down -v
Added a prominent warning in both the install and backup guides: `docker compose down -v` deletes the named volume containing the database. This is a common operator mistake.
