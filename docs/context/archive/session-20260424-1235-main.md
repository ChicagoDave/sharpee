## Key Decisions

### 1. Remediations hold — no rewrite
The 15 ADR-153 core decisions survived Phase 4 intact. The ten gaps identified in §3 of
the assessment were additive (new components not anticipated) or phrasing fixes, not
reversals of decisions that proved wrong. A rewrite would redo weeks of integration work
to clean up a few hundred lines of remediation cruft. Not worth it.

### 2. Addendum (ADR-153a) over overwrite
When offered the choice between amending ADR-153 in-place or writing a separate addendum,
the user chose the addendum pattern — matching how ADR-152 was preserved as a historical
record when ADR-153 superseded it. ADR-153a is explicitly cross-referenced from ADR-153
rather than silently replacing it.

### 3. Methodology rules live in CLAUDE.md, not ADRs
Two rules that emerged from the multiuser arc — 12a (Integration Reality) and 7b
(Co-Located Wire-Type Sharing) — are already in CLAUDE.md. ADRs cite them by rule number
rather than restating them. This kept ADR-156 focused on four load-bearing decisions
instead of sprawling into "everything about the client."

### 4. ADR-154 is taken
Partway through drafting ADR-155, the user flagged that ADR-154 is already assigned
to the Sharpee-IDE concept. New ADRs for this arc start at 155. (No ADR-154 was created
during this session.)
