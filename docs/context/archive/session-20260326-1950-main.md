### ISSUE-063 Phase 1: Trait-Access `as any` Cleanup

**Key discovery**: `getTrait` already has a generic overload accepting a trait constructor — `getTrait(OpenableTrait)` infers `OpenableTrait | undefined` automatically. No platform changes needed; the fix is purely caller-side: replace `getTrait(TraitType.X) as any` with `getTrait(XTrait)`.

**Bugs found and fixed during cleanup:**

1. `AuthorModel.setupContainer()` was setting `requiredKey` via `as any` — a property that does not exist on `LockableTrait`. The actual property is `keyId`. The cast was silently creating a runtime dynamic property with no effect on locking behavior. Fixed to use `keyId`.

2. NPC system had its own duplicate `Direction` type (lowercase: `'north'`, `'south'`) in `types.ts` separate from world-model's `DirectionType` (uppercase: `'NORTH'`, `'SOUTH'`). Removed the duplicate; NPC system now uses `DirectionType` from world-model.

3. `IExitInfo` was not exported from the room trait barrel (`packages/world-model/src/traits/room/index.ts`). Added the export so callers can use it without casting.

## Key Decisions

### 1. Constructor pattern over TraitTypeMap
The `getTrait` generic already supports passing a class constructor. No new platform types or maps were needed. The cleanup is a pure caller update.

### 2. Always choose correct fix over quick fix
User stated this preference explicitly (saved to memory). Applied throughout: `requiredKey` was not patched with a comment or suppression — it was corrected to `keyId` because that is the property that actually exists on `LockableTrait`.

### 3. `requiredKey` → `keyId` is a behavior fix, not just a type fix
Confirmed against `LockableTrait` definition before changing. The `as any` was silently creating a dead property, meaning any code relying on `requiredKey` in author setup would have been setting nothing.

### 4. Removed duplicate Direction type rather than casting
`npc/types.ts` defined its own `Direction` union. Removing it and importing `DirectionType` from world-model is the correct fix — type correctness and single source of truth.
