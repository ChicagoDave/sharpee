## Key Decisions

### 1. Typecheck Added to CI Before Build
The `tsc --noEmit` step runs on the umbrella `@sharpee/sharpee` package, which re-exports from all sub-packages. This catches cross-package type errors without requiring a separate per-package typecheck pass. Runs before the build step so type failures are reported quickly.

### 2. Five Packages Added to CI Test Coverage
The packages added (`event-processor`, `lang-en-us`, `if-services`, `text-service`, `transcript-tester`) were previously built by CI but not tested. Adding them closes the gap where a package could have passing types, a successful build, and broken tests — all invisible to CI.

### 3. Lerna Removed Rather Than Updated
Lerna was entirely vestigial — no scripts, no CI workflows, no build processes used it. Removing it was cleaner than updating and eliminated all 3 critical audit findings and 28 total vulnerabilities in one step.
