## Key Decisions

### 1. ADR-179 §First release version: `zifmia-v1.0.0`
The earlier `0.9.x` series tracks the pre-public build-from-source era and is not republished. `1.0.0` marks Phase 8 of ADR-177 + ADR-178 v1 as the audit-clear inaugural public release. Resolves the load-bearing OQ-1 from the first review pass.

### 2. ADR-179 §Tagging convention: `:edge` reserved for `main` pushes
Version-shaped tags (`<X.Y.Z>`, `<X.Y>`, `<X>`, `:latest`) are minted only by release-tag pushes. `main`-push events mint only `:edge`. Preserves Invariant 1 ("image is the release") without forfeiting the "what landed on `main` today" use-case.

### 3. ADR-179 §Multi-arch: free runners accepted at 25–45 min/release
Native ARM runners deferred until release cadence makes the wait painful — a one-line `runs-on:` change, no architectural lock-in.
