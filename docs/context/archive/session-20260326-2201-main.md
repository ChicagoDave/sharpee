### Phase 2A: Remove Legacy Event Sequencing Layer

The engine had a vestigial event-driven architecture layer that wrapped `ISemanticEvent` objects in `SequencedEvent` shells (adding `sequence`, `turn`, `scope` fields). No consumer used these fields. Platform listeners immediately reverse-engineered events back to `ISemanticEvent` using `as any` casts. This created approximately 25 `as any` casts in `game-engine.ts` alone, plus more in platform files.

**Approach**: Emit `ISemanticEvent` directly from the engine, delete the sequencing infrastructure entirely.

## Key Decisions

### 1. Preserve Event Enrichment Logic in turn-event-processor.ts
The `event-adapter.ts` contained normalization and enrichment logic (adding turn number, actor, location context to events). Rather than deleting this logic, it was inlined into `turn-event-processor.ts`. The enrichment is valuable — only the `SequencedEvent` wrapping was vestigial.

### 2. Simplify emitGameEvent() to a Direct Emit
`emitGameEvent()` in `game-engine.ts` previously wrapped events in `SequencedEvent` shells. It now simply emits the `ISemanticEvent` directly. All the `as any` reverse-engineering in platform listeners was deleted alongside the wrapping.

### 3. Platform Reconstruction Code Deleted Entirely
`cli-platform` and `browser-platform` contained code that reconstructed `ISemanticEvent` from `SequencedEvent` fields using `as any` casts. This entire reconstruction block was deleted in each file — the platform now receives the semantic event directly.
