## Key Decisions

### 1. Callbacks return SceneReaction, not raw strings
Callbacks return {text} or {messageId, params}; the plugin translates each to a game.message event. This means reactions are player-visible by construction — the prose pipeline's handleGameMessage already renders game.message events, so no new rendering path was needed.

### 2. timestamp:0 sentinel, no Date.now()
The monotonic counter already makes scene event ids unique. turn-event-processor.normalizeEvent backfills event.timestamp || Date.now() at ingestion time. Generating Date.now() inside scene event construction added no value and was removed for consistency.

### 3. Plan-review TENSION folded into test design
The plan-review advisory that end-to-end tests must assert on rendered blocks (not just emitted events) was honored: 3 of the 10 new tests drive the real ProsePipeline and assert on block content, directly preventing a recurrence of #151's silent-handler class of defect.
