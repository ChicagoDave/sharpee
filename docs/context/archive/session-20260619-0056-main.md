## Key Decisions

### 1. Defer TS7 adoption to TS7.1 (GitHub #131)
`@davidcornelson/tsf` depends on `ts.createProgram` (programmatic API), which TS7.0 withholds until 7.1. No adoption is possible without a tsf rewrite. Revisit at TS7.1 RC; decide between tsf port vs tsf-on-tsgo-CLI rewrite (the latter needs an ADR).

### 2. tree-sitter for IDE syntax highlighting (ADR-182)
Chosen over TextMate/Oniguruma (no incremental parse, no real type-awareness path) and CodeEditSourceEditor (not production ready as of 2026-06-19). SwiftTreeSitter 0.10.0 + tree-sitter-typescript.

### 3. In-code query string over bundled-query auto-discovery (ADR-182)
Bundled-query discovery (`Language.query(contentsOf:)`) fails silently from an app target due to bundle path resolution. In-code curated query is explicit, testable, and requires no resource-bundle registration.

### 4. Neon deferred — version conflict (ADR-182)
Neon main pins SwiftTreeSitter `branch:main`, incompatible with our pinned exact 0.10.0. Non-reproducible build risk outweighs marginal performance benefit on files of IDE scale. Revisit when Neon publishes a tagged release compatible with SwiftTreeSitter 0.10.x.

### 5. ADR-183 uniform span/class render model
All decoration types (inline, block, void) render as `<span class="sharpee-…" data-value="…">`. No semantic HTML on the wire. Layout is entirely stylesheet-driven. This keeps the renderer vocabulary-free and lets authors override any decoration's visual behavior via CSS without touching engine code.

### 6. Explicit-whitespace rule (ADR-183)
Source-text whitespace, including newlines, collapses to a single space. Authors use `[br]`/`[p]` for intentional breaks. This is consistent with ADR-174's decoration model and eliminates fragile array-join patterns in story source.
