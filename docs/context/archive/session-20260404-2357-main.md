## Completed

### ADR-142 (Conversation System) — All 5 Open Questions Resolved

1. **Topic resolution**: Exact-match keyword sets (alias-style), no stemming. Related topics handle near-misses so authors don't need to anticipate every phrasing.

2. **Conversation lifecycle and visibility**: Conversations are an active state with NPC intent (eager, reluctant, hostile, confessing, neutral) and strength (passive, assertive, blocking) driving between-turn commentary. Platform provides defaults; author overrides.

3. **Cross-NPC consistency reframed as attention management**: Conversations compete for player attention. Blocking strength can prevent the player from leaving, modeled analogously to a locked door on the going action.

4. **NPC-to-NPC conversation**: Two modes — offscreen (state mutations when player absent) and eavesdropping (player concealed, dialogue performed, knowledge asymmetry with source 'overheard'). Propagation and goal pursuit deferred to ADR-144 and ADR-145.

5. **Parameterized topics**: Author-defined params on response intents, resolved at render time by lang-en-us. Same pattern as existing language-layer parameterized messages.

Section 3 of ADR-142 was significantly expanded from "Conversation Flow and Momentum" to "Conversation Lifecycle and Attention Management." References section updated to include ADR-144, ADR-145, and ADR-146.

### ADR-144 (Information Propagation) — New ADR, All 3 Questions Resolved

- Author-defined propagation profiles per NPC: tendency (chatty/selective/mute), audience (trusted/anyone/allied), content control (whitelist/blacklist), coloring (dramatic/vague/fearful/etc.), pacing (eager/gradual/reluctant), scheduling via character state conditions
- Player-directed propagation: player can intentionally use an NPC as a messenger (tell the gossip to spread a fact)
- Provenance tracking: source chain on every fact
- Three visibility modes: offscreen, witnessed, eavesdropped
- Resolved: author defines which propagated facts create topics (#1), author defines whether received info is fact vs. belief (#2), author controls whether NPCs propagate lies or truth (#3)

### ADR-145 (NPC Goal Pursuit) — New ADR, All 4 Questions Resolved

- Authored behavior sequences gated on conditions, not AI planning
- Goal activation from character state; step types: seek, acquire, waitFor, moveTo, act, say, give, drop
- Three pursuit modes: sequential, opportunistic, prepared (the Clue killer pattern)
- Goal interruption and preemption by higher-priority goals
- NPC traversal of locked passages is narrative, not mechanical: NPC simply goes through, player sees a witnessed message, door remains locked for the player
- Resolved: author-defined movement profiles with map knowledge and access (#1), observation via existing ADR-141 system (#2), combat takes over from goal's act step (#3), no cap on simultaneous active goals (#4)

### ADR-146 (NPC Influence) — New ADR, No Open Questions

- Simple binary system: NPC exerts influence, target resists or does not
- Author defines everything: influence names (arbitrary strings), effects, resistance conditions, messages — no platform taxonomy
- Conditional resistance with except clauses (character-revealing tells: "Margaret resists seduction except from women")
- Influence as a goal pursuit step: Colonel intimidates gardener to gain access to the study
- Duration vocabulary: while present, momentary, lingering
- PC influence creates narrative gameplay moments: focus clouded, context disrupted

## Key Decisions

### 1. Author-First Design Across All Four ADRs

No derived scores, no inferred behavior, no platform taxonomies. The platform provides scheduling and dispatch; the author defines all semantics. This keeps the engine general while making story logic explicit and readable.

### 2. Character State as Scheduling Mechanism

All four ADRs use character state (from ADR-141) as the scheduling primitive rather than implicit turn counters or timers. This keeps the system coherent — one place to look for why an NPC does something.

### 3. Attention Management as the Right Frame for Conversations

Reframing "which NPC gets priority" as "conversations compete for player attention" solved the cross-NPC consistency question cleanly and produced a design that composes with existing action validation patterns.

### 4. NPC Locked Passage Traversal Is Narrative

NPCs moving through locked doors is a narrative event, not a mechanical one. This avoids adding a second-class locking exemption system and keeps the mechanical world model honest: locks apply to the player.

### 5. Clean Composition Chain

The four ADRs compose in a directed chain:
- Conversation (ADR-142) leaks information
- Propagation (ADR-144) spreads it between NPCs
- Goal pursuit (ADR-145) acts on it
- Influence (ADR-146) modifies character state that gates all of the above

All four build on top of ADR-141 (Character Model) without requiring changes to it.
