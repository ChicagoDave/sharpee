## Key Decisions

### 1. `[SKIP]` skips command execution, not just assertion
Discovered that `[SKIP]` suppresses the entire command turn — navigation commands inside `[SKIP]` blocks never ran. All navigation that set up test preconditions was moved out of `[SKIP]` blocks and given real assertions.

### 2. `[CONTAINS_ANY]` is not valid syntax
Correct form is `[OK: contains_any "a" "b" "c"]`. Any use of the standalone `[CONTAINS_ANY: ...]` directive produces a parse error.

### 3. `[DO]`/`[UNTIL]` not available in published tester 0.9.95
The syntax exists in source but was not included in the published npm package at 0.9.95. Transcripts 10 and 14 that relied on it were rewritten using a fixed number of sequential wait/attack commands, which is sufficient given combat randomness averages.

### 4. `registerBasicCombat()` needs a try-catch guard
The combat interceptor registry is global across the Node process. Calling `registerBasicCombat()` twice in the same test run throws. Wrapped in `try { registerBasicCombat() } catch {}` in `initializeWorld()`.

### 5. MoveableSceneryTrait does NOT block taking
Unlike `SceneryTrait`, `MoveableSceneryTrait` does not set `portable: false`. The metal panel transcript was adjusted to omit a "cannot take" assertion.

### 6. Save/restore and undo produce no output in transcript testing mode
Both commands run without error but emit no text. Transcripts 45 and 46 were written to assert `[OK]` (command accepted) rather than inspecting text content.
