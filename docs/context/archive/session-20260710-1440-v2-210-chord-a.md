## Post-Phase-6: Chord formatting grammar (David-approved, same session)

David flagged the quoted/prose mix in cloak.story as flaky and approved four grammar changes (first entries in docs/architecture/chord-grammar-changes.md, all dated 2026-07-10):

1. **Prose block is the only phrase-text form** — same-line quoted AND bare values removed (`parse.phrase-text-form` with fix-it). Quotes remain for header strings + hatch paths.
2. **Blank line inside a prose block = paragraph break** (`\n\n` in IR text). In create blocks the block still ends at a blank line (keeps keyword lines out of prose); consecutive bare paragraphs append to the description (duplicate-description error removed).
3. **`{br}` built-in marker = hard line break**; `br` reserved as phrase/hatch name (`analysis.reserved-marker`); whitelisted by the unbound-marker gate.
4. **`verbatim` phrase modifier** (comma slot, exclusive with strategies/`or`) — line structure, interior blanks, and relative indentation preserved (common indent stripped); IR gains optional `verbatim: true`.

Implementation: chord lexer untouched (afterBlank already carried); parser (parseProseParagraph paragraph continuation in strictlyAbove mode, parseVerbatimBlock terminating only at column-1 lines, description append, same-line rejection with recovery); analyzer (br whitelist + reserved checks, verbatim flow); loader mapping rides EXISTING platform machinery — engine prose pipeline `createBlocks` splits `\n\n`→paragraph block / `\n`→tight block, so descriptions/templates need only real newlines; `{br}`→`\n` at registration (src/text.ts withLineBreaks, absorbs joiner spaces); verbatim phrases register `{verbatim:text}` (ADR-158 template parser's whitespace-exempt atom) with the exact text supplied as an emit-time param — this even preserves leading indentation end-to-end in the CLI.

**Second latent loader bug found by the formatting probe story:** the interceptor registry is keyed (traitType, actionId), so two entities with `on reading it` clauses had the second registration silently clobber the first (cloak never noticed — one reading target). Fixed: one dispatching interceptor per action routing by target entity (runtime.ts buildDispatchingInterceptor); state-asserting test in story-loader tests/formatting.test.ts.

Fixtures/tests/docs updated: cloak.story dequoted (fixture + story copy + design.md §3.1), ac5-random, 3 gate fixtures, 2 malformed fixtures, runtime.test.ts inline sources; snapshots regenerated (verified: 3 quoted→prose + 1 verbatim:false, rest span-only); chord formatting.test.ts (11 tests) + story-loader formatting.test.ts (4); ac3-cloak-sweep unbound-marker line 91→92; chord-grammar.md EBNF + prose/formatting section; design.md phrase-form rationale amended.
