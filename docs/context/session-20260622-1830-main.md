## Key Decisions

### 1. Web-native renderer pattern for invented channels
An invented channel's renderer must `createElement` its own container and `appendChild` to a stable platform element (`#main-window`). It must NOT guard an element that the author adds to the host page — the host page is platform-owned and regenerated. This pattern mirrors `TextDisplay` appending to `#text-content` and is now the canonical Ch 25 example.

### 2. ADR-187 accepted — split-first, no shared code
`@sharpee/devkit` will be author-only (outside-repo, published). In-repo platform builds move to a new `tools/repokit` (unpublished). The two are completely separate — no shared modules, no cross-dependency, overlap is duplicated. Rationale: repokit is the proving ground; features port to devkit when hardened. Accepted before implementation begins; the split deserves its own focused session.

### 3. `./sharpee` wrapper stays, points at devkit
The `./sharpee` root wrapper continues to work for authors running `sharpee build dungeo`. A new `./repokit` wrapper serves in-repo platform work. `./sharpee build dungeo` routes to `./repokit build dungeo` post-split. No author-facing CLI change.
