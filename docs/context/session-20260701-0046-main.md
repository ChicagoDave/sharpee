### 2. Melee combat blank-output fix

Root cause: `stories/dungeo/src/messages/npc-messages.ts` registered four dynamic melee message IDs (`dungeo.melee.hero_attack`, `dungeo.melee.villain_attack`, `dungeo.melee.backup_weapon`, `dungeo.melee.villain_disarmed`) as empty-string templates. Under Phrase Algebra, `getTemplate('')` returns `''` (not undefined), so `renderViaPhrase` produced a blank non-null block that shadowed the inline-text fallback the melee interceptor relies on, producing blank combat output. Fix: removed those four empty registrations; kept `STILL_RECOVERING` and `UNARMED_ATTACK` which have real text. Unregistered IDs now fall through to the inline-text path (same behavior as `dungeo.melee.villain_recovers`, which already worked).

Secondary fix in `stories/dungeo/src/combat/melee-messages.ts`: `getHeroAttackMessage` only replaced `{villain}`, leaving `{the villain}` as a literal string. Fixed to replace `{the villain}` → `"the <name>"` (longer form first) and `{villain}` → `"<name>"`.

Result: wt-01 went from 32 passed / 99 failed → 36/36 pass.

### 3. Low Room carousel fix (MDL-canonical)

Root cause of the wt-10→wt-17 cascade: commit `a6f1e970` had inverted intent for the Low Room. Verified against MDL source (`docs/internal/dungeon-81/original_source/act1.254 CAROUSEL-EXIT` and `act3.199 MAGNET-ROOM-EXIT + TRBUT`): the single `CAROUSEL-FLIP!-FLAG` drives the two rooms oppositely — the triangular button toggles the flag TRUE, which *fixes* the Round Room but *randomizes* the Low Room (`<PROB 50>` → Machine Room or Tea Room). So `robot-entity.ts:104` setting `carousel.active = true` is MDL-correct; the bug was that the wt-10 return path traversed the Low Room after the button push without a retry loop.

Fix: kept `robot-entity.ts` at `active = true` with a corrected comment; wrapped wt-10's Low Room return path in a `WHILE`-loop retry to Tea Room (exactly like the existing Round Room `WHILE` loop in the same file). Left `Math.random()` in `carousel-handler.ts` as-is (matches `round-room-handler.ts`; deterministic RNG is future work). Corrected the backwards Low Room note in `stories/dungeo/CLAUDE.md`.

Result: fixed wt-10, which cascaded to fix wt-11 through wt-17 (all inherited broken save state via `$restore`). Full chain: 402/648 → 885/885 pass.

## Key Decisions

### 1. MDL-canonical Low Room behavior over always-deterministic

Chose to preserve the MDL's `<PROB 50>` randomized Low Room exit (after button push) and handle it in the transcript via a `WHILE`-loop retry, rather than patching `carousel-handler.ts` to always exit deterministically. User direction: verify against MDL source and match its intent.

### 2. Deterministic/seeded RNG deferred

Replacing `Math.random()` in carousel and round-room handlers with a seeded RNG (for fully-deterministic transcripts) is deferred as future work. User noted the project has never successfully gotten transcripts to work with deterministic fake randomization, and the retry-loop approach is sufficient.
