## What happened

- **Straggler scan (ADRs 192–204).** Found: (1) the phrase-algebra *atoms* are complete and
  clean — no legacy colon-chain formatter remnants, no live `entityInfoFrom` calls, `npcName`
  fully gone; (2) the one real loose end — **subject-verb agreement was only applied to dialogue
  + NPC attribution (ADR-201/203), never the general action catalog**. ~120 templates across ~25
  action files still hardcode a present-tense verb after a `NounPhrase` subject
  (`{the target} collapses`), so a plural subject would mis-agree.

- **ADR-205 drafted** (catalog-wide subject-verb agreement). `/adr-review` → **7/12 NEEDS WORK**:
  the review caught that this is a **two-axis** migration — templates *and* every emitting
  action must pass its subject as a number-carrying `NounPhrase` (`nounPhraseFor`), else `{verb:}`
  silently degrades to singular. A one-axis template rewrite would be a no-op. Plus a structural
  fence would be needed to stop the pattern recurring.

- **The strategic question** ("are we going to keep bouncing between breaking and fixing?") was
  answered with an **incidence count** instead of another point-fix:
  | Source | Plural entities | Plural NPCs/targets |
  |---|---|---|
  | dungeo (flagship) | 0 | 0 |
  | friendly-zoo | 3 (goats/rabbits/parrots) — all `.scenery()` | 0 |
  | tutorials/familyzoo | same 3 (duplicates) | 0 |
  | platform defaults | 0 | 0 |
  The friendly-zoo plurals are non-portable scenery appearing only in room/slot descriptions
  (already on the migrated ADR-195/203 path), not action targets. Real-world incidence of the
  gap ADR-205 fixes: **~0**.

- **ADR-205 REJECTED** and rewritten as an evidence-based rejection record with an **on-demand
  fallback** (migrate only the specific templates + emit-sites an entity hits, if a story ever
  introduces a plural action-subject; if it ever gets frequent, do it once — both axes + fence).
