### TypeScript 6 Upgrade Risk Assessment
- Sharpee is on TS 5.9.3 (declared `^5.2.2`)
- Assessed risk: mostly safe. `tsconfig.base.json` already explicitly sets the options TS 6 changes defaults for (`esModuleInterop`, `allowSyntheticDefaultImports`, `moduleResolution: node`, explicit `target`/`module`, `rootDir`, `strict: true`)
- Main risk identified: TS 6 changes `types` default from auto-include to `[]`; packages relying on implicit `@types/*` pickup will break silently
- Code-side risks assessed as low: 7 declaration-merging uses unaffected, 1054 `as any` usages unaffected, 5 `@ts-expect-error` trivial to handle
- Recommendation: wait. TS 5.9 is stable; TS 6.0 is a transitional release prepping for TS 7.0 (the Go-port rewrite with major speed improvements). Migrate when TS 7 ships and the compile-time speedup becomes relevant.

## Key Decisions

### 1. TypeScript Is a Feature, Not a Liability
The session clarified and locked in the rationale for Sharpee's language choice. Dynamic TS features (structural typing, runtime flexibility, dynamic event payloads) are specifically beneficial for IF authoring — they are not a compromise. Future sessions should not revisit this as an open question or propose ports as a performance optimization.

### 2. Do Not Rush a TS 6 Migration
TS 6 is a bridge release toward TS 7 (Go port). Migrating now pays a compatibility cost without receiving the primary benefit (compile speed). The codebase is already well-insulated against most TS 6 default-change breakage, so the migration when TS 7 arrives will be low-risk.
