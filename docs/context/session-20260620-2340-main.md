## Key Decisions

### 1. chain handler event type (drift from per-version docs)
`chainEvent` must return a custom event type with a `text` field. Returning `game.message` is wrong for chain handlers ‚Äî the processor treats it as an action-text override. Documented prominently in ch13 and cross-referenced in ch22.

### 2. Capability dispatch is always single behavior + hand-written action in v14
The registry enforces one behavior per trait+capability. v14 uses a single `pettingBehavior` with an internal `animalKind` switch, plus a hand-written dispatch action. `createCapabilityDispatchAction()` exists in stdlib but is not used in the tutorial ‚Äî noted as a shortcut in the chapter.

### 3. Preface comparison principle
Comparisons to other IF systems are acceptable when direct and rational; punching-down language ("can't do this", "fall off a cliff") is not. TADS dropped from comparisons because its single-language model doesn't support the same contrast.
