## Key Decisions

### 1. Browser template lives entirely in `@sharpee/devkit`
The `@sharpee/sharpee` umbrella package knows nothing of browser scaffolding; `ts-forge.json` `assets: []` on that package was already correct. No change to the sharpee npm package.

### 2. Runtime deps added to the author's `package.json` (not pulled into the umbrella)
`init-browser` injects the 5 platform-browser deps directly with `platformRanges()` pinning. The rejected alternative (pulling `platform-browser` into the `@sharpee/sharpee` umbrella) would have widened the engine-level API surface unnecessarily.

### 3. CSS is platform-owned, not seeded
`base.css` / `decorations.css` / `styles.css` / `themes/` / `index.html` are copied fresh from devkit on every `build-browser`. The author's sole customization surface is `browser/<story-id>.css`, linked last so it wins the cascade.
