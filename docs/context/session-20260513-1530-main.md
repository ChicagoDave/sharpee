## Key Decisions

### 1. pnpm deploy --legacy for runtime bundle
`pnpm 10` requires `--legacy` for non-injected workspaces. Without it, the deploy step fails to resolve symlinked peer deps inside the monorepo.

### 2. Inline ESM-extension rewrite in Dockerfile build stage
`tsc -p tsconfig.esm.json` with `moduleResolution: "bundler"` emits extensionless imports that Node ESM rejects. The host build works only because `tsf` was previously run. Inside a clean Docker build this surfaces immediately. Fixed with an inline Node script that walks every `dist-esm/` tree and rewrites 138 imports across 25 packages. This is a platform-level fragility — not a Docker-specific issue — flagged in ADR-178 follow-ups.

### 3. ADR-178 scope: manifest package owns the baseline list
The in-session patch (adding 5 `@sharpee/*` deps + `lz-string` to `tools/zifmia/package.json`) is the short-term fix. ADR-178 formalizes the long-term answer: a manifest package holds the canonical list; zifmia declares a single dep on that package; story bundles are validated against it at load time.
