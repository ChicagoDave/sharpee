## Key Decisions

### 1. actions.yaml as Single Source of Truth
Rather than requiring authors to call `entity.annotate()` directly, all editorial overrides (suppressions, custom labels, caps, category assignments, conditions) live in a single `actions.yaml` file in the story directory. The build compiles this into annotation data. This keeps the authoring surface clean and the annotation API internal.

### 2. Build Flag Gate for Tree-Shaking
The entire computation module is guarded behind `globalThis.INCLUDE_CONTEXT_ACTIONS`. When the flag is absent from the build command, the constant is `false` and esbuild tree-shakes the entire module from the bundle. Text-only clients (CLI, Zifmia) are zero-overhead.

### 3. ISSUE-058 as Documentation, Not Platform
Entity creation helpers do not need to be stdlib/world-model additions. A guide document with copy-paste patterns serves the same purpose without adding API surface or maintenance burden to the platform.
