## Key Decisions

### 1. Landing page is a discovery surface, not a lobby

Rooms remain passcode-gated. The listing is a discovery signal only. Only rooms with at least one connected participant appear (accepts the "flicker" trade-off for simplicity).

### 2. PH authors the room title

Set at create time, shown on the public landing page. 80-character cap, non-empty required. Editable later via PH settings drawer.

### 3. Passcode modal, not a dedicated join page

Single-page-app feel. Clicking a room on the landing page opens a modal asking for the passcode, rather than navigating to a separate URL.

### 4. All four Zifmia themes ship in v0.1

classic-light, modern-dark, retro-terminal, paper. Per-user preference via a header button, global scope (not per-room), persisted in `sharpee.theme` localStorage key. Theme button lives in one shared component visible in both landing and room headers. Theme work is in Plan 01 rather than deferred to Plan 05.

### 5. Terminology: "multi-user" / "multiuser", never "multi-player"

MPIF (Multi-Player IF) is a distinct IF concept reserved for a possible future product. All docs, UI strings, and code use multiuser.

### 6. Single accessible-first client; no sibling text client in v0.1

Accessibility bar is Frotz-parity SR experience. Screen-reader smoke moved into Plan 01 Phase 10 (AC1 blocker) rather than deferred to Plan 05. State layer kept UI-agnostic as an architectural hedge so a future sibling client is cheap if blind-player feedback asks for one.

### 7. Client lives at `tools/server/client/`, not `packages/`

`tools/` is the standalone monorepo-external zone for product-coupled infrastructure. The browser client is tightly coupled to this one server and belongs there. This also prevents it from appearing in the library package dependency graph.

### 8. Node server serves `dist/` as static assets with SPA fallback

This directly closes the KEY FINDING from session-20260422-0508-main.md (`GET /` returning 404 on the Ubuntu deployment). The server gains a `GET /api/rooms` endpoint and a static-file + SPA-fallback middleware in Plan 01 Phase 2.
