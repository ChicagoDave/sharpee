## Key Decisions

### 1. v1 line pinned via `origin/v1` branch (not a tag)
The 1.x line at `sharpee_v1` is active maintenance, not a frozen point-in-time. A branch allows commits; a tag would not. This keeps the two repo checkouts cleanly separated: `sharpee_v1/v1` for 1.x maintenance, `sharpee_v2/main` for the v2 platform.

### 2. 12-member Phrase union declared upfront
David chose to declare all 12 kind interfaces in Phase 1 — the 5 foundational (`Literal`, `NounPhrase`, `PhraseList`, `Sequence`, `Empty`) fully implemented, and 7 reserved stubs (`Pronoun`, `Numeral`, `Verbatim`, `Contents`, `Slot`, `Optional`, `Choice`) that throw `PhraseNotImplementedError` at runtime. Rationale: the ADR-192 prose treats the union as closed; an additive-5 design would require revisiting the discriminated union definition in a later ADR and risk narrowing mistakes at call sites.

### 3. v2 branch-per-phase workflow
`main` is maintenance-only from this point forward; all phrase-algebra work lives on `v2` (long-lived integration branch). Each phase gets its own named branch (`v2_phase1`, etc.) that merges into `v2`. Nothing ships to `main` until ADR-192 is complete. This is codified in plan.md and in memory file `v2-branch-workflow.md`.

### 4. Plan-review contradiction fixes
(a) `{list:items}` is NOT a valid kind-prefix in the new grammar — messages using it are re-authored programmatically in Phase 3, not passed through as a `list:` kind. (b) Phase 1 creates the `v2` branch in-place (`git checkout -b v2` in this checkout) rather than a `git worktree add` at the occupied path; D3's worktree-add prescription assumed a different primary-checkout layout.

### 5. `{list:items}` re-authoring path
Because `list:` is not among the recognized kind prefixes (`pronoun:`, `number:`, `contents:`, `slot:`, `verbatim:`), any action that formerly used `{list:items}` in its message template must instead build a `PhraseList` programmatically in the report layer and bind it to a param key that the template references as a bare `{items}` or named NounPhrase.
