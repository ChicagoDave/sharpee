## Key Decisions

### 1. Version subdirectories under docs/book/

Chose `docs/book/v1.5.0/` and `docs/book/v2.0.0/` over sibling top-level directories. Keeps all editions under one tree; simpler BOOK_DIR resolution in the build script; mirrors the `docs/reference/` convention already used in the repo.

### 2. Duplicate assets into each edition

Each edition is self-contained — no shared image or asset references across editions. Storage cost accepted in exchange for isolation: one edition can freely add/remove/rename assets without breaking the other.
