## Completed

### Arc 1 — System 6 line-height (GH #111)
Diagnosed FindersKeepers font metrics (WinAscent+WinDescent = 0.791em, USE_TYPO_METRICS unset): at 24px with `line-height: 1.5` the per-line leading was ~17px — nearly as tall as the glyph, reading as double-spaced. Tightened `--theme-line-height` from 1.5 → 1.0 in `templates/browser/infocom.css` for `:root[data-theme="system-6"]`. Commit `99ddf536`.

### Arc 2 — Bold room name via `[room:...]` decoration
Discovered `room_description` template in `lang-en-us/src/actions/looking.ts` (and `going.ts`) is dead code — the real rendering path is `packages/engine/src/prose-pipeline/handlers/room.ts:51`, which emits `ROOM_NAME` and `ROOM_DESCRIPTION` blocks directly without touching those templates. Fix: wrap `resolvedName` with `[room:${resolvedName}]` so `parseDecorations` produces a `sharpee-room` IDecoration, rendered as `<span class="sharpee-room">` (already styled `font-weight: bold` in `decorations.css`). Commit `e441e136`.

### Arc 3 — Remove `white-space: pre-line` (full refactor)
User audited 75 `\n`-embedding locations across packages and stories and chose the full scope. Changes:
- `ITextBlock` gained `tight?: boolean` in `packages/text-blocks/src/types.ts`.
- New `createBlocks(key, text)` helper in `packages/engine/src/prose-pipeline/assemble.ts` splits on `\n\n+` (paragraph boundary) and `\n` (line boundary). Each `\n` becomes a block boundary; continuation blocks carry `tight: true`. No block content ever holds a `\n`.
- All engine prose-pipeline handlers (generic, about, domain-message, room, help, command-failed, game, audibility, client-query, revealed) converted from `[createBlock(key, text)]` to `createBlocks(key, text)`.
- Wire shape updated in `@sharpee/if-domain`: `TextContent[][]` → `MainEntry[]` (`{ content, tight? }`). Producer (`stdlib/src/channels/standard.ts`) and consumer (`platform-browser/src/channels/main.ts`) both updated. Renderer applies `main-entry--tight` CSS class for tight entries.
- `templates/browser/base.css`: added `.sharpee-prose-pane p.main-entry--tight { margin: 0 }`.
- `white-space: pre-line` deleted from `platform-browser/src/channels/main.ts` and `platform-browser/src/display/TextDisplay.ts`. `TextDisplay` now handles its own `\n`-splitting internally.
- `scripts/bundle-entry.js` accepts both old and new wire shapes for backward compatibility.
- Plan written to `docs/work/pre-line-removal/plan.md` and committed. Commit `517a0274`.

### Arc 4 — Structured banner with semantic CSS classes
User observed non-tight `<p>` margins (0.6em) still caused banner spacing to read nearly 2x. Implemented per-piece CSS classes:
- `ITextBlock` and `MainEntry` gained `className?: string`. Plumbed through `createBlock` opts, `mainChannel` producer, and browser renderer (applied as additional CSS class on `<p>`).
- New `packages/engine/src/prose-pipeline/handlers/banner.ts` exports `buildBannerBlocks(key, story, engineVersion, context)`. Emits structured pieces in order: `game-title`, `story-version`, `platform-version`, `sub-title`, `author-list` (one block per credit entry), `banner-spacer` (empty `<p>` for visual gap), then optional `game.banner.story-tail` template appended via `createBlocks` for story-specific footer text.
- `handleGameStarted` and `handleAboutDisplayed` both delegate to `buildBannerBlocks` — identical structured banner for game-start and ABOUT command.
- `StoryConfig` (engine), `GameEventStoryData` (core), `StoryInfoTrait` (world-model), and `AboutDisplayedEventData.params` (stdlib) all gained `credits?: string[]`.
- `templates/browser/base.css`: added zero-margin rules for each banner class; `banner-spacer` has `min-height: 1em`; `game-title` defaults to bold.
- `stories/dungeo/src/index.ts`: populated `credits: ["By Tim Anderson, Marc Blank, Dave Lebling, Bruce Daniels", "Ported by David Cornelson"]`; threaded through `StoryInfoTrait`.
- `stories/dungeo/src/messages/index.ts`: removed multi-line `game.started.banner` override; added `game.banner.story-tail` entry.
- Room handler: first description block after a room name is marked `tight: true` so it sits flush under the bold name.
- `engine/tests/prose-pipeline/handlers/about.test.ts` and `game.test.ts` rewritten to assert on structured className shape.
- GitHub issue #112 (ABOUT showing nothing) fixed as a side effect — ABOUT now calls `buildBannerBlocks` directly.
- Deployed to `~/public_html/dungeo/` (live at https://sharpee.net/play/dungeo/).
