## Key Decisions

### 1. ADR-185 — IDE as standalone authoring tool only
- The IDE never builds the platform. Platform is always a prebuilt npm dep.
- `sharpee introspect` is the introspection entry point (devkit CLI, not a separate node script).
- No `--skip` change tracking in the IDE; no monorepo mode.

### 2. tsf publish strips `files` array — use `ts-forge.json` `assets`
- Any non-`dist` content that must ship through `tsf publish` requires `"assets": ["<glob>"]` in the per-package `ts-forge.json`. This is the canonical fix, not a workaround. Saved as memory.

### 3. Browser template base: `@sharpee/platform-browser`, framework-free, minimal
- David confirmed: base browser template should be framework-free channel-io via `@sharpee/platform-browser` (ADR-170 settled this). No audio in the base template (audio becomes a separate "Add audio" IDE capability later).
