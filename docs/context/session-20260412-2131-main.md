## Key Decisions

### 1. Extension location: tools/vscode-ext/ not packages/
The extension is developer tooling, not a platform package. It has no place in the pnpm workspace dependency graph and should not be registered with the 6-point new-package checklist. `tools/` is the right home.

### 2. CodeLens for "Run Goal"
VS Code does not support clickable gutter buttons. CodeLens (inline above the line) is the correct affordance for per-goal test invocation. The "Run Transcript" lens appears above the first command in the file.

### 3. esbuild for bundling
esbuild produces clean single-file bundles in ~2ms. No Webpack complexity needed for an extension this size.

### 4. CLI path via configuration
`sharpee.cliBundlePath` setting defaults to `dist/cli/sharpee.js` but is user-configurable, making the extension portable across machines with different repo layouts.

### 5. New Story Wizard uses inline templates
Files are scaffolded directly from template strings in `new-story-wizard.ts`. There is no `npx @sharpee/sharpee init` command yet; adding one is future work.
