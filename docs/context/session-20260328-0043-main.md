## Key Decisions

### 1. Capture at event time (Option A)
Trait snapshots are captured in the runner when events are captured, reflecting world state at that exact moment. The reporter just formats what it receives. This avoids post-hoc state reconstruction and ensures snapshots are accurate to the turn they represent.

### 2. Deep recursive string extraction
`collectStrings()` recursively walks event data to find entity IDs in arrays (like `itemIds`) and nested objects, not just top-level values. This ensures traits are captured for all affected entities, not just the primary target.

### 3. Prose for CLI, raw for pipeline
The CLI always shows prose when `--emit-traits` is on. The parse-walkthrough parser handles both JSON and prose formats. The website renders prose from the parsed data. This keeps the CLI output human-readable while the pipeline retains structured data for future use.

### 4. --emit-traits implies --verbose
Trait emission requires verbose event output to be meaningful, so the flag automatically enables verbose mode. Keeps the flag surface clean.
