### Browser Build Fix
- `./build.sh -s dungeo -c browser` was failing because esbuild in `--platform=browser` mode only resolves `browser`, `default`, `import`, and `module` export conditions, but the package.json `exports` field only had `types` and `require` conditions.
- Fix: Added `--conditions=require` flag to the esbuild browser bundle command in `build.sh` (line 826).
- Also fixed a macOS-specific `sed -i ''` syntax to Linux `sed -i` in the HTML template copy step.

### Copy-to-Website Script
- Created `copy-to-website.sh` — copies `dist/web/dungeo/*` to `website/public/web/dungeo/` and `dist/stories/dungeo.sharpee` to `website/public/downloads/dungeo.sharpee`.

## Key Decisions

### 1. Use `--conditions=require` Rather Than Modifying package.json Exports
The package exports only define a `require` (CJS) condition. Rather than adding new export conditions to every package's `package.json`, the esbuild browser bundle command was given `--conditions=require` so it resolves the already-defined CJS entry points. This is a minimal, targeted fix with no downstream impact.
