## Key Decisions

- **Closure-per-channel model** (ADR-163): each channel carries its own `produce`
  closure rather than a shared rule-based router. Reduces coupling, enables
  per-channel capability gating.
- **Wire types in if-domain**: `IOChannel`, `ChannelDefinition`, wire packet shapes
  (`HelloPacket`, `CmgtPacket`, `TurnPacket`, etc.) live in `@sharpee/if-domain`
  so engine and renderer can share them without circular deps.
- **ChannelService over text-service in browser**: R5-C fully replaces the
  `text:output` path in BrowserClient with `channel:packet` dispatch.
  `@sharpee/text-service` peer dep removed.
- **ADR-165 renderer is in channel-service**: the `Renderer` class and
  `ChannelRenderer` interface live in `packages/channel-service/src/renderer/`
  rather than a separate package — consistent with the "ship a small renderer"
  scope in ADR-165.
