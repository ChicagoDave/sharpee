## Key Decisions

### 1. Integrated-Progressive Book Structure
Single continuous narrative with skippable asides rather than separate Author / Programmer volumes or separate books. Reasoning: lowers barrier to entry while not hiding the technical side. Extensions and ADR rationale are carved to a future book so the author book stays focused.

### 2. Pandoc Single-Source Toolchain
Local script (no CI), markdown source → EPUB + PDF + HTML. User requirement was ebook + printable output from one canonical source.

### 3. Programmer's Track Scope Constraint
The Programmer's Track covers only the public `@sharpee/*` API — classes, methods, type signatures — from the installed npm package. No implementation bodies, no monorepo `src/` paths, no ADR citations in the book itself.

### 4. "Tracks" Not "Paths"
Reserved "path" for API/import paths throughout the book. The two reading perspectives are Author's Track and Programmer's Track.
