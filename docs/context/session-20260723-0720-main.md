### 1. `/play` browser client — fixed on plover (DONE, verified)
Two-layer root cause. First, `./repokit` runs the compiled `tools/repokit/dist/cli.js`, which was stale on plover (dated Jun 30) — last session's ADR-252 Chord-aware fix (skip `package.json` read for `.story` stories, `build.ts:104`) was correct in source but never recompiled here, and `pnpm -r build` halts at a pre-existing friendly-zoo `isAlive`/`INpcData` compile error before reaching devkit/repokit. Fixed with a cold rebuild: `pnpm install` -> `pnpm -r build` (platform packages green) -> `pnpm --filter "@sharpee/repokit..." build` (devkit + repokit) — recorded to project memory as `[[plover-cold-build]]`.

Second — the actual platform bug: `mirrorToWebsite()` in `tools/repokit/src/commands/browser.ts` copied only a hardcoded subset of files (`game.js`, `index.html`, three engine CSS files) into `website/public/web/fernhill/`. It omitted `story.story` (the client does `fetch("./story.story")` and compiles it at boot — this is fatal without it), `fernhill.css` (linked by `index.html`, would 404), and the `audio/`/`images/` runtime asset directories. David approved the fix as a platform change (CLAUDE.md gate honored) before implementation: rewrote `mirrorToWebsite` to clear the target directory and recursively copy the entire devkit `outDir` (`cpSync` recursive), guaranteeing byte parity with the author build instead of hand-maintaining a file list.
