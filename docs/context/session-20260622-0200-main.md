## Key Decisions

### 1. Book teaches against the decoupled standalone tutorial
The tutorial consumes published `@sharpee/*@^1.0.8` (not in-repo path aliases). The monorepo build is for the platform only. This makes every book code example verifiable by a reader who does exactly what the book says: `npm install`, then run.

### 2. Parrot: static pettable actor (ch15) → full NPC (ch20)
Resolves the chapter-vs-version NPC ordering conflict and fills ch20's missing parrot in a single design move. No invented story content — David confirmed both the ch15 static form and the ch20 promotion.

### 3. SceneryTrait taught at ch4
Introduced where it first appears in the zoo story rather than deferred to a later chapter. Keeps code examples consistent with what readers can actually run.

### 4. classifyStory reads package.json, not path
Path-derived package naming was fragile (any directory rename would silently mis-classify). Reading the real `package.json` name is unambiguous; `workspace:*` detection is the correct signal for "managed by pnpm".
