`eventBuffer` is fully populated when `produceTurnPacket` runs.

## Key Decisions

### 1. Pattern B (channel-service in CLI consumer, not engine)

The plan's stated intent (Phase 3 retires text-service's wire role *in the CLI path*; Phase 4
handles platform-browser) only fits Pattern B. Pattern A would require simultaneously migrating
browser, transcript-tester, and the engine itself — beyond Phase 3's budget. With Pattern B,
engine still instantiates text-service inside `game-engine.ts:224` and continues to call
`textService.processTurn(turnEvents)` per turn. The CLI consumer wraps the produced blocks with
channel-service rather than rendering them directly.

### 2. `text:output` Hook (not `turn:complete`)

`turn:complete` carries `result.blocks` and `result.events` together but does **not fire for
meta-commands** (VERSION, SCORE, HELP, RESTART) — meta-commands take an early-return path through
`executeMetaCommand` that skips the `turn:complete` emit. Switching exclusively to `turn:complete`
would cause meta-command output to silently disappear.

`text:output` fires for both normal and meta paths. The CLI consumer pairs it with the existing
`engine.on('event', ...)` accumulator (`eventBuffer`) to recover the events alongside the blocks.
By the time `text:output` fires, the engine has already emitted every event for that turn, so
`eventBuffer` is complete.

### 3. Plain Text in CLI (No ANSI Decorations)

`flattenContent()` strips decoration wrappers. CLI output is plain text in Phase 3. Documented
trade-off in the plan: ANSI / terminal colors are a Phase 4+ concern, layered as terminal-only
formatting on top of `main` channel content. AC-13 passes with plain output, so no regression
from a test perspective.

### 4. Walkthrough Chain Convention Codified

The "GREEN = any of N runs passed" convention was implicit in the project but not durably
recorded. Updated `feedback_flakey_walkthroughs.md` to spell it out, including the cross-link
to GH #106 (largest flake contributor). Future sessions will not re-litigate this.

### 5. cli-platform.ts Out of Scope

`packages/platforms/cli-en-us/src/cli-platform.ts` exists but `bundle-entry.js` does not use it.
It is a separate consumer path, not on the AC-13 critical path. Phase 3 documented its status;
its fate (delete or migrate) is deferred.
