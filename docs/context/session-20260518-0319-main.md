### Diagnosed silent tsbuildinfo no-op bug in pnpm clean path
`pnpm clean` runs `rimraf dist dist-esm` but does not touch the 47 `.tsbuildinfo` files sitting next to each `tsconfig.json` (not inside `dist/`). On rebuild, `tsc` sees stale tsbuildinfo and decides inputs are up-to-date, emits nothing with exit 0. build.sh prints `[core] ✓` falsely. Three packages later `@sharpee/if-domain` fails with TS2307 "Cannot find module '@sharpee/core'" because `dist/` was never recreated. Pre-existing, unrelated to tsf; only bites after `pnpm clean`.
