## Key Decisions

### 1. Complete codebase separation — no shared modules
R1 of ADR-187: overlap (consumer-gen, test/play stubs, introspect/ifid) is duplicated, not shared. Both tools depend only on published libraries (`@sharpee/bootstrap`, `@sharpee/transcript-tester`, `tsf`). This means repokit can iterate without risk of regressing published devkit consumers.

### 2. Repokit as smoke-test proving ground
Features are proved in repokit (in-repo, fast feedback) and ported by hand to devkit. The one-class-per-file Command contract is the portability boundary — no internal abstraction leaks across.

### 3. Walkthrough RNG flakiness is pre-existing and orthogonal
The Dungeo walkthrough chain has cascading grue-death flakiness from combat RNG when an unlucky early death cascades kills. This predates the ADR-187 split; `wt-01` passes standalone. One clean deterministic pass is authoritative (David's framing). A separate issue to seed combat for the walkthrough regression baseline is warranted but orthogonal to this work.

### 4. Header-comment polish deferred
Repokit files copied from devkit retain `@sharpee/devkit` in their owner-context header comments. A hand-fix pass is needed in a later session. NOT safe to sed-replace because `repo.ts` contains `'@sharpee/devkit'` as a package-name string.

### 5. Incidental artifact discipline
`./repokit build dungeo` regenerates `packages/sharpee/docs/genai-api/*` and stamps `stories/dungeo/src/version.ts`. These must be reverted before committing unrelated work. Noted as a session lesson.
