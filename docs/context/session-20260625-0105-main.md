### Generator: `scripts/build-snippet-page.cjs`
Parses all book chapters (docs/book/parts/**/NN-*.md, in book order, skipping volume-divider H1s). For each chapter:
- Extracts raw code steps (typescript/bash/css/jsonc fenced blocks; `under the hood` heading flags blocks as `reference` type).
- Pairs with the runnable cumulative file from `tutorials/familyzoo/src/chNN-*.ts` by chapter number.
- Chapters without a matching tutorial file carry the previous chapter's cumulative forward ("no story-code change this chapter"); ch01 shows "no cumulative yet."
- Chapters 24–28 (presentation/multi-file) carry ch23 forward because their tutorial code is in directories rather than single files.

Produces one standalone HTML page (embedded CSS/JS). Features: 525px fixed left-panel TOC with scrollspy active-highlight, copy buttons on every snippet block, light/dark theme toggle. Palette mirrors `site/style.css` exactly: dark default with champagne-gold accent `#f2d280`; light mode uses blue `#3b82c4`. Toggle persists via `localStorage['sharpee-theme']`, the same key as `site/theme.js`. Page title = "The Sharpee Author and Developer Manual" throughout. Uses `__dirname`-based `ROOT` constant so the script is path-independent. Optional args filter to a subset of chapters (e.g., `node build-snippet-page.cjs ch02 ch04`).

Output: `docs/book/code-snippets/index.html`.

## Key Decisions

### 1. Page Organized by Book Chapter, Not Tutorial Version
The natural unit for readers is the book chapter they are reading, not the tutorial version file. This drove the chapter-first traversal (docs/book/parts/**/NN-*.md) rather than iterating tutorial checkpoints.

### 2. Carry-Forward for Chapters Without Tutorial Checkpoints
Chapters 1 and 24–28 have no single-file tutorial checkpoint (ch01 has none yet; ch24–28 use multi-file directories). Rather than showing a gap or an error, the page carries the previous chapter's cumulative forward with an explanatory note. Future work can wire in multi-file chapters.

### 3. Delta + Cumulative Pairing on Each Chapter Page
Each chapter shows both: the raw step-by-step deltas (what the book presents) and the full runnable checkpoint (what the reader would have after completing the chapter). This was the design David stated when correcting the flat-file approach.
