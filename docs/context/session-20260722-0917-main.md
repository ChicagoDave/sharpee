## Key Decisions

### 1. ADR-254 supersedes ADR-231 D1b — dots illegal everywhere, no carve-out
Verified against real code (no shipped `.story` used dotted platform keys); the only consumers were synthetic tests, so removing the dotted spelling costs nothing real. See ADR-254 amendment and ADR-231 D1b SUPERSEDED pointer.

### 2. ADR-255 ACCEPTED — message-override capability restored via an ACL
The dotted spelling is gone but the override capability stays behind a curated kebab anti-corruption layer (Interface Contract 3, mirroring `catalog.ts`'s existing Interface Contract 2): `override message <alias>` / `override messages <locale>`, full `define phrase` parity, 734-entry hand-curated alias catalog, loader-side completeness pinning (total bijection against the live `lang-en-us` message-id set), story-wide-default precedence (on-clause refusal → per-entity phrase → override message → platform default).

### 3. Phase 1 scoped to author-label sites only; event types deferred to ADR-256
Rather than block Phase 1 on a fresh design question, applied the established framework (author label → kebab-migrate now; platform-bound id an author shouldn't hand-type → curated translation) and shipped Phase 1 with `allowDotted: true` at the 3 event-type sites. This mirrors ADR-254's own precedent of keeping message-override alive behind ADR-255 rather than breaking a capability before its replacement exists.

### 4. ADR-256 ACCEPTED — story-loader gains a Chord-id → platform-id event translation
No runtime translation exists today (verified, not assumed — see Discovered mid-implementation above); ADR-256 adds one in `@sharpee/story-loader` so Chord stays platform-free while platform event ids stay dotted. Author-defined events kebab on both sides (no map entry needed); only platform-owned event families (`media.*`) get explicit map entries. This finishes ADR-254's ban uniformly once Phase 1b lands.

### 5. Process correction — apply an established framework instead of re-litigating it
Mis-designed the ADR-216 collision fix three times (curated ACL, then mechanical rename-the-platform, then a false "already translates" claim) before David corrected each. Root causes: over-engineering when a mechanical answer was correct, and asserting how the codebase works from a quick grep instead of tracing the actual call path. Captured in memory as `apply-established-framework.md` (updated this session) — apply a settled rule to each new instance without re-opening it as a fresh menu; verify call paths before asserting; prefer the mechanical/simple fix over building new machinery.
