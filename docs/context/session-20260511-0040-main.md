## Key Decisions

### 1. better-sqlite3 over node:sqlite
Already in `onlyBuiltDependencies` and used by dungeon-walk; no new native-module surface area. Synchronous driver wrapped in `Promise.resolve` keeps the `StorageAdapter` interface async and backend-identical.

### 2. Fastify over Express
Chosen for Phase 3's AC-14 performance budget (<100 ms p95 on a 2-vCPU SQLite container). Fastify's request lifecycle and schema-based serialization are the margin.

### 3. In-process FIFO lease for SQLite
`Map<roomId, DeferredLease>` implements single-writer guarantee per room within a single process. Postgres advisory lock (`pg_try_advisory_lock`) is deferred to Phase 7 as the multi-process equivalent.

### 4. Story bundles stored as DB blobs
`story_bundles` table holds bundle content. ADR-175 §Decision 3 invariant: "not on container filesystem." Phase 5 will enforce this invariant through the admin upload path.

### 5. pnpm workspace integration
`tools/zifmia` added to `pnpm-workspace.yaml` and root `package.json`. Workspace:* deps allow local package references without publish cycles during development.

## Progressive update — Phase 3a complete (2026-05-11)

**Phase 3a — Identity + Session Middleware** — first sub-phase of Phase 3 per the commit-cadence decision.

**Status**: COMPLETE.

**Tests**: 46/46 passing (up from 23 at Phase 1 close). New tests added:
- 5 storage contract tests for session methods (`createSession` FK violation, `getSessionByToken`, `deleteSession` idempotent, `deleteExpiredSessions` filters)
- 5 scrypt passcode tests (round-trip, rejection, random salt, malformed-hash no-throw, format)
- 8 identity-route tests including **AC-11**: `expect.toEqual({error: 'invalid_credentials'})` for both wrong-passcode and nonexistent-handle paths — body identical
- 5 auth-middleware tests (missing header, missing prefix, unknown token, expired session, success path)

**Smoke test** confirms AC-11 end-to-end: `curl POST /identity/login` with bad passcode and with nonexistent handle both return `{"error":"invalid_credentials"}` 401.

**Design choices recorded:**
- **scrypt over bcrypt/argon2.** `node:crypto` built-in, no native dep beyond what better-sqlite3 already brings. OWASP 2023 interactive parameters (N=32768, r=8, p=1). Self-describing hash format `scrypt$N$r$p$salt$key` lets future tuning bumps verify older hashes without migration.
- **Random 32-byte hex session tokens** stored in `sessions` table (256 bits entropy). No JWT, no HMAC token — DB lookup per request is fine for v1; simpler operational story; supports revocation trivially.
- **OQ-2 resolved**: env var name is `ZIFMIA_WORKER_COUNT` (binds at Phase 3e).
- **AC-11 same-body extends to validation failures.** Malformed body also returns 401 `invalid_credentials` rather than 400 — wire observers cannot distinguish "wrong passcode" from "malformed JSON." Documented inline in the route.
- **Variance from ADR-161 noted.** ADR-161 specifies server-generated `(Id, Handle, passcode)` triples; ADR-175's route table shows user-supplied passcode at login. v1 Zifmia takes the simpler "user picks passcode at registration" flow. Documented in `src/server/identity.ts` header.
