## Key Decisions

### 1. Front-Matter Pandoc Ordering Fix
Pandoc's auto-TOC is injected at the start of the body when writing HTML/PDF, so input-files listed in `book.yaml` land after the TOC. Solution: `frontmatter.yaml` prepends input-files for EPUB (order of `--defaults` args controls input-file order); `--include-before-body` with a pre-rendered HTML fragment handles HTML/PDF where the template places `include-before` ahead of the TOC. This avoids any custom pandoc template changes.

### 2. EPUB Redundant Title Heading
EPUB readers inject an `h1` from `metadata.title` for the first file if it has no heading. The authored `.book-title` span was therefore a duplicate. Fix: author the title page without an `h1`; suppress `.book-title` in `epub.css`. EPUB gets the injected heading; HTML/PDF get the authored span.

### 3. Art Masters Gitignored; Optimized Outputs Committed
Original Met downloads are large and would bloat the repo. Masters live in `art/_masters/` (gitignored, local). `build-book.sh` generates committed `art/*.jpg` from masters only when stale. Any contributor who needs to re-optimize must obtain the masters separately (URL list in `art/art-list.md`).
