### Root cause of Docker build failure identified
`zifmia-publish.yml` triggered an `:edge` build that failed at `pnpm --filter '@sharpee/zifmia...' run build` with Rollup unable to resolve `@sharpee/platform-browser/channels/main`. Root cause: `packages/platform-browser` was the only one of 20 workspace packages whose `build` script was `tsc` alone — no `tsconfig.esm.json`, no `dist-esm/` output. Its `package.json` `exports` map already pointed import-condition entries at `./dist-esm/channels/<name>.js`, but those files never existed.

### Path A precursor implemented and verified
Added `packages/platform-browser/tsconfig.esm.json` (mirrors `packages/engine/tsconfig.esm.json`, with `lib: ["ES2022", "DOM", "DOM.Iterable"]` override). Updated `packages/platform-browser/package.json`: `build` to `"tsc && tsc -p tsconfig.esm.json"`, `clean` to `"rm -rf dist dist-esm"`. Both `pnpm --filter '@sharpee/platform-browser' run build` and `tools/zifmia/pnpm build:web` now succeed. The Rollup resolution error is gone.

### Path B scoped and blocked

Attempted `pnpm add -Dw tsf` to bring tsf in as a devDep; pulled an unrelated npm package (`tsf@0.0.7` by user "qinjia"). Rolled it back. User will publish their tsf under their org (likely `@chicagodave/tsf`) and return to complete Path B.
