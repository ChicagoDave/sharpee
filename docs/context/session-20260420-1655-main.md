## Key Decisions

### 1. Clock Injection in LockManager
Rather than calling `Date.now()` directly in the lock manager, the `createLockManager` factory accepts a `Clock` interface (`{ now(): number }`). This removes wall-clock dependency from tests and satisfies the AFK-timer mock requirement in the Phase 5 definition of done.

### 2. Snapshot Isolation for getState
`getState` returns a frozen object copy rather than a reference to the internal map value. This prevents handler code from accidentally mutating lock state outside the manager's API surface, making the state machine easier to reason about under concurrent handler calls.

### 3. Vendored Zip Before Node SEA
Node SEA is the correct long-term distribution format but requires a signed binary per platform and a stable build pipeline. Vendored zip (bundled JS + bundled runtime + story files in a zip) unblocks distribution immediately without committing to a binary signing workflow. ADR-154 IDE work may justify revisiting SEA when the IDE ships.

### 4. Plugin Clarification in ADR-154
Early IDE mock drafts placed plugins as an IDE project type alongside Story and Extension. During review, it became clear that plugins (in the Sharpee sense) are runtime objects registered with the engine, not editor-time artifacts. The IDE's Plugin project type was removed; the dependency panel shows which plugins a story depends on, but the IDE does not scaffold or edit plugin implementations.
