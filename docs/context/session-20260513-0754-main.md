## Key Decisions

### 1. Each spec spawns its own server for isolation
`globalSetup` only preflights that the required build artifacts exist; it does not start a shared server. Each spec file spawns and tears down its own `node dist/main.js` process on a free port. This ensures test isolation at the cost of ~5 processes per run ‚Äî acceptable at 31 tests total and ~5s suite runtime.

### 2. Negative WS assertions via marker frames
When verifying "no broadcast happened," a follow-up frame is sent from another client and asserted to arrive before any disallowed frame rather than using a fixed timeout. This avoids slow/flaky timeout-based negative assertions.

### 3. ZIFMIA_GRACE_MS env var for AC-5 clock override
Rather than injecting a clock mock into the server, AC-5 passes `ZIFMIA_GRACE_MS=200` as an env var to the spawned process. This exercises the real grace-period path at test-friendly speed without any stub.

### 4. Handle regex constraint discovered in test output
The `/^[A-Za-z]{3,12}$/` constraint (letters only, 3-12 chars) caused several initial failures. Test handles were adjusted to conform; no server-side change was needed.
