## Key Decisions

### 1. Option B: Narrow `--allow-read` Deno Permissions

Spawn command becomes `deno run --allow-read=<bundle>,<story> --v8-flags=--max-old-space-size=256 <bundle_path>`. Not `--allow-none` — the bundle needs read access to itself from disk. Not `--allow-read=.` — too broad. The container is the outer boundary; these flags are the inner boundary against rogue story code. ADR-153's "no filesystem access" language means no access to the server's stateful surfaces, not zero file descriptors ever.

Alternatives rejected:
- **Option A (stdin stream)**: would require redesigning the wire protocol's startup sequence.
- **Option C (hybrid)**: unnecessary complexity given the container provides the outer boundary.

### 2. Option D1: Server-Side Install-Time Pre-Bundle

When a `.sharpee` file lands in `/data/stories/`, the server: unzips it, invokes esbuild programmatically to combine the host entry + `story.js` + all transitively-resolved `@sharpee/*` packages into a single self-contained ESM, writes to `/data/stories-compiled/<slug>.host.js`, keyed by source mtime. On subsequent requests, spawns Deno against the cached compiled bundle.

Alternatives rejected:
- **D2 (Docker-build-only)**: operators can't install new stories without rebuilding the image.
- **D3 (per-spawn bundle)**: esbuild runs on every room spawn — too slow, defeats sandbox startup latency goal.
- **D4 (import maps)**: Deno import maps don't resolve Node/pnpm workspace paths without additional infrastructure.
- **D5 (change `.sharpee` format)**: breaks the existing build pipeline for no gain.

### 3. Explicit Plugin Enumeration in the Bundler

The bundler hand-imports the known plugin set (`plugin-npc`, `plugin-scheduler`, `plugin-state-machine`) rather than auto-discovering them from `story.js` imports. "Simple, honest" over "clever, fragile." If a new story needs a new plugin, the bundler list is updated explicitly. In practice, the story imports its own plugins transitively anyway, so the hand-maintained list is a belt-and-suspenders measure, not a hard requirement for v0.1 dungeo.
