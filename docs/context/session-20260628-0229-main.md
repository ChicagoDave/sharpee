## Key Decisions

### 1. `createHelpers(world)` over `world.helpers()` monkeypatch for bundle compatibility
`WorldModel.prototype` monkeypatching does not survive esbuild bundling because each package gets its own inlined world-model copy. Explicit `createHelpers(world)` is the correct v2 form; dungeo already avoids helpers entirely. No platform change â€” `createHelpers` already exists in `@sharpee/helpers`.

### 2. NounPhrase `name`/`number` contract: `name` IS the plural surface for `number: 'plural'`
For an intrinsically-plural entity, `.plural()` sets the entity name to the plural form directly; `nounPhraseFor` maps `IdentityTrait.name` straight through with no singular available. The Assembler must not re-pluralize `name` when `number: 'plural'`. `pluralForm` remains the field for an explicit plural override of a singular-named entity. Documented in ADR-192 after the NounPhrase interface definition.

### 3. ADR-195 ACCEPTED: realize-time Slot core; turn-time channel deferred
Core design: `Slot { key, fallback? }` kind + `slotContributions?: Map<string, Phrase[]>` optional read accessor on `RenderContext`. Stories contribute via a `SlotContributor` hook registered on `ProsePipeline.processTurn` (realize-time only). Turn-time `contribute()` channel deferred to a follow-on â€” the optional seam is declared in the interface but not implemented until needed. Implementation via `v2_adr195_slot` branch; additive (no red-build cutover â€” Slot stub currently throws, no template uses it).

### 4. Phase 2 of ADR-195 plan requires platform-change discussion
The engine store (`ProsePipeline.registerSlotContributor`, per-turn `RenderContext` slot map) touches `packages/engine` â€” a platform package. Flagged in the plan and must be discussed before that phase begins per CLAUDE.md conventions.
