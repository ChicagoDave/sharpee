## Decisions (David, 2026-07-12, this session)

1. **`must be any <name>` PARSES** (P2 flag 2 resolved): membership
   semantics — the subject must satisfy the named open condition.
   `<subject> must be any <open-cond>: <key>` compiles to a new IR
   condition kind `{ kind: 'satisfies', subject, condition }` — a dated
   deviation from proposal §3.5's pinned IR contract (any-of/none-of/
   each/match only), to be reconciled in the P5 ratchet-log pass.
   `must be no <name>` is a negated requirement → error (D6 stance,
   same as `must not`).
2. **`match` is a reserved name** (P2 flag 3 resolved): declaring an
   entity named `match` (or alias), a trait data field `match`, or a
   grammar slot `:match` is a load error (`analysis.reserved-name`) —
   the `br` precedent. Multi-word names containing "match" stay legal.
