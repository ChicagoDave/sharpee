## Key Decisions

### 1. Region IDs Are Author-Supplied
`createRegion()` accepts an explicit string ID (e.g., `'reg-underground'`) rather than auto-generating one. Used direct `IFEntity` construction + `entities.set()` instead of `createEntity()` to avoid the auto-prefix behavior. This matches the authoring intent: region IDs must be stable and meaningful for assignment and traversal.

### 2. Scene Conditions Are Closures Stored Outside SceneTrait
Begin/end conditions are stored in a `SceneConditions` map keyed by scene entity ID. This map is separate from the trait so closures are never serialized. After save/restore, stories must re-register condition closures. Trait data (state, turn counts) persists normally through serialization.

### 3. Scene Evaluation Is a Built-in Plugin
`SceneEvaluationPlugin` is auto-registered by `GameEngine` at priority 60. Stories never need to register it. This matches the pattern for the NPC turn plugin — built-in platform behavior, not story configuration.

### 4. Region Events Fire After actor_moved
`if.event.region_exited` fires (innermost-first) then `if.event.region_entered` fires (outermost-first), both after the `actor_moved` event in the going action's report phase. This ordering ensures event handlers can assume the actor's location is already updated when they receive crossing events.

### 5. Phase 8 Skipped
Dungeo room-to-region assignment was deferred by user request. No existing Dungeo tests are affected; the `regionId` field on `RoomTrait` is optional.
