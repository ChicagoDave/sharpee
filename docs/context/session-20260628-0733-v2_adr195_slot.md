## Key Decisions

### 1. `ITextService` → `IProsePipeline` as a prerequisite commit
Rather than routing `registerSlotContributor` around the transitional ADR-174 interface name, David chose to do the rename first as a standalone commit. This resolves the naming debt ADR-174 anticipated and keeps the interface name honest now that the engine controls it exclusively.

### 2. Slot owns all connective grammar; contributions are bare content
Commas, conjunctions, and sentence breaks are assembled by `renderSlot`, not by contributors. This ensures correct punctuation for any contribution count and avoids a second punctuation authority. The existing `joinParts` function is reused for clause-mode joining to keep the authority count at one.

### 3. Turn-time channel named-and-deferred
The realize-time `SlotContributor` hook (Phase 2) stages before the event→render loop. The per-event/turn-time channel (ADR-163/S4) is explicitly deferred and documented in the engine tests; it is not a gap in Phase 2.

### 4. Phase 4 `'detail'` key is single-shared per turn
The first Friendly Zoo consumer will use a single `'detail'` slot key per turn (examine targets one object). Per-entity keyed slots are a named follow-on if multi-object description in one turn becomes real.
