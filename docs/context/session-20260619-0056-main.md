## Completed

### TypeScript 7.0 RC adoption research (committed, merged to main)
- Researched the TS7.0 native (Go) compiler RC, which shipped **2026-06-18**: ~10x typecheck speed (VS Code 77s → 7s), `tsgo` binary, new `--checkers`/`--builders`/`--singleThreaded` flags.
- **Key finding — the blocker is the toolchain, not Sharpee's source.** `./sharpee build` runs `@davidcornelson/tsf` (v1.0.0), which is built on the TypeScript *programmatic API* (`ts.createProgram` in `compilers/tsc.js`, `compilers/esbuild.js`, `transform/declarations.js`, `rollup-bundler.js`). TS7.0 ships **no stable programmatic API** — Microsoft deferred it to **7.1** (months out). Bumping tsf's `typescript` dep to 7.0 will not work (same breakage class as ts-morph).
- **Sharpee source is nearly clean**: no decorators in `packages/`, all TypeScript (no `.d.ts`-from-JS), targets fine (ES2022 / commonjs / es2022). Two config migrations remain whenever adoption happens: `moduleResolution: "node"` → `"bundler"`/`"nodenext"` (one inherited base change), and `baseUrl` → `paths` in ~10 tsconfigs.
- **Recommendation: defer to TS7.1.** The speed win is real but gated on exactly the API 7.0 withholds. The only way to get it sooner is a tsf-on-`tsgo`-CLI rewrite — a platform change needing an ADR.
