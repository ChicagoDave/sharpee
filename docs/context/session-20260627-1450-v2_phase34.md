- **§7.3 (LanguageProvider API) resolved** as `renderMessage(messageId, params, ctx)`
  with the orchestration (perspective→parse→realize) living inside lang-en-us, not the
  engine. The engine only supplies the per-turn `RenderContext`. Matches the scope-doc
  signature; keeps language realization in the language layer.
- **`WorldModelLike` minimal interface** in render-context.ts instead of importing the

  public-hoists `@sharpee/*` (pnpm-workspace `publicHoistPattern`), which is how the
  existing assembler already imports text-blocks undeclared. Declaring it forced a
  lockfile regeneration; reverted to match the existing convention and keep frozen
  install working. (Candidate cleanup later: declare all real `@sharpee/*` deps.)
