## Completed

### The question, answered by reading source

Chord's **award** half is complete: `score <name> worth N` on four owners (story header, `create`,
`define trait`, `define action` — `chord/src/ast.ts:76`, `ir.ts:469`, `chord.ebnf:156`), `award`
resolving owner-first, and maxScore derived as the sum of every declared `worth`
(`story-loader/src/loader.ts:518`).

The **rank** half does not exist. `rank` appears nowhere in the lexer, parser, AST, IR, EBNF, or
language reference — only inside message-alias names.

Four verified defects behind that gap:

1. `computeRank()` (`stdlib/src/actions/standard/scoring/scoring.ts:52-60`) returns the English
   literals `Novice`/`Amateur`/`Proficient`/`Expert`/`Master` at fixed percentage bands — a standing
   language-layer violation independent of Chord.
2. `scoring.ts:87`/`:99` read `enabled` and `rank`; **neither field is in `ScoringCapabilitySchema`**
   (`stdlib/src/capabilities/scoring.ts:11-42`). Both branches unreachable through the declared
   contract.
3. The schema constant reaches **no production path**. `StandardCapabilitySchemas`
   (`capabilities/index.ts:54-61`) feeds `registerStandardCapabilities()`, called only from
   `stdlib/tests/test-utils/index.ts:34`, `capability-refactoring.test.ts`, and
   `engine/tests/integration/query-events.test.ts:32`. **stdlib's action tests therefore exercise a
   capability-present path no shipped story has** — which is why the dead branches went unnoticed.
   Dungeo registers SCORING directly with `{ moves, deaths }` (`stories/dungeo/src/index.ts:233`),
   bypassing the schema.
4. Consequence: a story with no scores answers SCORE with *"Your score is 0 points"*; the honest
   `no_scoring` branch can never fire. `rank_*` and the whole `scoringSystemMessages` block
   (`lang-en-us/src/actions/scoring.ts:27-31`, `:60-78`) are emitted by nothing.

**ADR-129 half-superseded ADR-085 and left the remainder in the tree** — score and maxScore moved to
the ledger; rank, enabled, achievements, and moves stayed in a capability nothing registers.

### ADR-260 written — "Scoring is a trusted extension; the platform keeps only the ledger and a thin SCORE" (ACCEPTED)

D1–D7. Supersedes ADR-085 and ADR-076 outright. Deletes the capability rather than repairing it;
consolidates on the ADR-129 ScoreLedger; guts `if.action.scoring` to a ledger reader; deletes eight
of eighteen scoring messages (the five `rank_*` plus the four progress messages — all percentage-band
prose stdlib had no business inventing) and prunes both alias tables in parallel; adds
`@sharpee/ext-scoring` shaped like `registerBasicCombat`, with an `if.event.rank_risen` TurnPlugin.

Design settled by reading, not assuming:
- `TurnPlugin` is `{id, priority, onAfterAction, getState, setState}` — cannot own a verb, and is
  documented not to run for meta-commands, which SCORE is. So "scoring as a plugin" could not own
  SCORE.
- `registerBasicCombat` contributes **no verbs** — ATTACK stays in stdlib, the extension supplies an
  interceptor and a resolver. That precedent is what made "scoring as an extension" work with
  SCORE-the-verb staying put.

### ADR-261 written — "Chord gains `use scoring` and a `ranks` ladder" (ACCEPTED)

D1–D8. `use scoring` as the third `EXTENSION_REGISTRY` entry; a `ranks` block under it; the gate
covering `score`/`award`/`ranks` together; generic loader lowering; per-rung `says <phrase>` for
promotions; and the Chord version bump with its enforced EBNF pin.

### Interview and three review passes

`adr-interview` resolved four questions (two collapsed into one shared decision). `adr-review` ran
three times and each pass found real defects:

- **ADR-260, 7/16 → 16/16.** Four blockers: `registerScoring(world, {ranks})` was unreachable through
  `registerWorld: (world) => void` on a module-level const map (and the workaround was banned by the
  ADR's own acceptance criterion); D7 contradicted D3 on un-migrated dungeo's SCORE output; "ranks
  persist for free" was false against `ScoreLedger.toJSON()`'s fixed two-field literal; and the
  schema was live in test infrastructure the Context claimed it wasn't.
- **ADR-261, 9/16 → 16/16.** Three blockers: `says` had no channel to any renderer (neither
  `IRRankDef` nor `RankDefinition` carried a key); the Chord version machinery was unnamed despite
  being test-enforced; and nothing asserted the fifteen-file migration.
- **Three-ADR pass (259/260/261): one blocker that survived a 16/16.** ADR-260 D6 placed the
  `registerPlugin` call site beside the `registerWorld` loop — impossible, since no plugin registry
  exists at world-build time. Every loader-registered plugin lives in `onEngineReady`
  (`loader.ts:679`, invoked from `game-engine.ts:515`). Corrected in both ADRs.

## Key Decisions

1. **Scoring becomes a trusted extension, not stdlib and not a bare plugin** — ADR-260 D5. Owner
   proposed plugin, then extension; the extension model fit because combat proves extensions layer
   behind stdlib verbs.
2. **The capability is deleted, the ledger is the single home** — ADR-260 D1/D2.
3. **Rank thresholds are absolute points, with a stated invariant that a maxScore change never moves
   a rank boundary** — ADR-260 D2. Forced by dungeo raising the ceiling 616 → 650 at runtime on the
   thief's death (`melee-interceptor.ts:210`); under percentages that call would silently *demote* a
   player who earned nothing and lost nothing.
4. **Ranks are configuration, not saved state** — ADR-260 D2. `ScoreLedgerData` unchanged, so
   existing saves load without migration.
5. **The gate covers `score`/`award`/`ranks` together** — ADR-261 D4. Fifteen `.story` files migrate.
6. **Ships as Chord 1.1 — a recorded one-time override of ADR-257 D2**, which defines a construct
   that no longer compiles as a *major*. Owner ruled 1.1 rather than amending the rule or softening
   the gate; cross-noted at ADR-257 D2 so the exception is discoverable from the rule.
7. **No platform prose for promotions at all** — ADR-261 D7. Per-rung `says <phrase>`, rendered by a
   loader-registered story reaction keyed on the rank id; a rung without `says` is silent.
