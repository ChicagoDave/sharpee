## Design notes / decisions

- **`realize()` block key**: Phase 1 fixed the signature `realize(tree, ctx): ITextBlock[]` with no key param. Phase 2 emits a single block under `ASSEMBLER_DEFAULT_BLOCK_KEY` (= `action.result`); the report layer assigns real channel keys in Phase 4. No Phase-1 contract change.
- **Definite vs indefinite list**: the phrase model carries article choice on each `NounPhrase`, so `PhraseList` needs no `mode` field — ADR-190's `{list}`/`{the-list}` split falls out of item `articleType`.
- **Transient duplication, by design**: the old `formatters/list.ts` / `article.ts` / chain still exist and still drive the *current* pipeline (Phase 4 rewires it). The Assembler reproduces the logic; Phase 3 deletes the old formatters. Not an oversight.
- **Count formatter singular surface** ("one coin") is genuinely Numeral territory → ADR-198 stub. Phase 2 covers the plural count-grouping ("three coins", "two geese", "ten"/"11 goats") through `PhraseList`; the "one X" surface is deferred.
