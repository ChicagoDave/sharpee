## What was done (3a)

- **`if-domain`**: added additive optional `NounPhrase.capitalize?: boolean` — the carrier for the `{capitalize …}` hint (ADR §5 gap; Phase 1 NounPhrase had no carrier). Language-neutral request, not a surface string.
- **`lang-en-us`**:
  - `src/parser/parse-phrase-template.ts` — `parsePhraseTemplate(template, params)`: literal segments → `Literal`; bare/hinted placeholders → `NounPhrase` (hints `the`/`a`/`an`/`some`/`capitalize`); `kind:` heads → reserved stub kinds; **rejects** legacy `:`-chains (AC-8), unknown kind-prefixes, unknown hints, and unbound params (AC-11) — all at parse time. `PhraseParseError` carries `offendingToken` / `template` / `reason`.
  - Assembler Case authority wired: `renderNoun` applies `capitalizeSentenceStart` when `np.capitalize`.
  - Param-binding model: value already a `Phrase` → used (NounPhrase hints override `articleType`/`capitalize`); string → indefinite `NounPhrase` default (per plan); number/boolean with no hint → `Literal`.
- **`stdlib`**: `src/utils/noun-phrase.ts` — `nounPhraseFor(entity, ctx)`, the ADR §3 `IdentityTrait → NounPhrase` mapping. `referableId`←`entity.id`; legacy `article` literal deliberately NOT mapped (Assembler computes the surface). `ctx` reserved for computed names/state (ADR-193+). No `IdentityTrait` → minimal indefinite singular noun.
