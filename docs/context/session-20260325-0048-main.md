- On the 4th PA announcement, the `zoo.after_hours` world state flag is set
- A one-shot departure daemon removes the zookeeper from the world and emits a farewell message

  - Rabbits complain about the quality of their pellets

- Parrot NPC behavior swaps at runtime from daytime squawking to after-hours articulate personality via `npcService.removeBehavior()` + `npcService.registerBehavior()` — the canonical Option B pattern described in the plan

## Key Decisions

### 1. Multi-file organization by author concern, not code type
Organized files around what a zoo story author thinks about (the map, the characters, the events, the score) rather than what a programmer thinks about (traits/, actions/, daemons/, npcs/). This mirrors how Inform 7 extension authors structure work and is more intuitive for newcomers.

### 2. Option B for parrot behavior swap (canonical pattern)
Used `npcService.removeBehavior()` + `npcService.registerBehavior()` rather than a conditional branch inside the existing behavior's `onTurn`. This is the "runtime behavior swap" pattern the plan designated as V17's showcase concept. The simpler Option A (branch on world state) is documented in comments as the tradeoff.

### 3. After-hours dialogue is one-shot per animal
The after-hours daemon tracks which animals have spoken using a closure-level Set. Each animal speaks exactly once per playthrough. This prevents repetition while keeping the daemon simple (no persistent state in world-model).

### 4. Keeper departure uses one-shot daemon with existence check
The zookeeper is removed from active play when after-hours activates. The one-shot departure daemon checks for zookeeper existence before firing to avoid double-removal.
