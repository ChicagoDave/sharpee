## Key Decisions

### 1. Structural gate as Vitest test, not ESLint rule
No custom ESLint plugin infrastructure exists in the repo. ADR-202 AC-1 explicitly permits "a lint rule OR test." A failing Vitest test reds CI with identical effect and avoids introducing ESLint plugin build machinery. The TS compiler AST used in the test provides accurate enclosing-function scoping — more reliable than regex over source text.

### 2. Dungeo build verification deferred
`./repokit build dungeo` has a pre-existing stale-workspace blocker unrelated to this session: `if-domain` dist is from Jun 26 (predates `Slot.mode`/`conj`), `packages/bootstrap` has never been built so repokit's tool won't compile. User confirmed this is a known follow-up; dungeo build verification deferred from both phases.

### 3. Phase 2 merge pending user direction (resolved)
Phase 2 was merged to main alongside Phase 3 as part of the continued session; resolved before Phase 3 work began.

### 4. Quote does NOT own the leading comma (Phase 4)
In templates like `says, "{quote:topic}"`, the comma before the opening glyph stays the template's responsibility. This avoids double-comma and prose-scanning to deduplicate; AC-4 holds via the literal template comma. Confirmed with user before coding.

### 5. Default top-level position is sentenceInitial:false (Phase 4)
No regression to existing messages — auto-cap triggers only inside Sentence/Quote or when `ctx.position` says so. Confirmed with user before coding.

### 6. Localized first-run marking over position threading (Phase 4)
Sentence/Quote mark the first run of their own realized child; `realize()` honors `ctx.position`. Chosen over threading a position param through all 13 `realizeToRuns` cases — observably equivalent, far lower-risk. Confirmed with user before coding.
