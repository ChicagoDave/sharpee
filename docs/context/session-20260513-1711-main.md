## Key Decisions

### 1. Root `.npmrc` with `public-hoist-pattern[]=@sharpee/*` and `lz-string`
Dynamic-imported `story.js` lives in `tools/zifmia/.bundle-cache/`. Under pnpm strict isolation, Node ESM walks up from that directory and only sees zifmia's direct deps — manifest-transitive packages (`@sharpee/media`, `lz-string`, etc.) are unreachable. Hoisting makes the dev workspace match the flat `node_modules` that `pnpm deploy --legacy` produces in Docker. This is the idiomatic pnpm fix for transitive-node_modules access in monorepos.

### 2. Root devDep on `fflate`
The `scripts/validate-bundle-baseline.js` validator extracts `story.js` from `.sharpee` zips and lives outside any package boundary. `fflate` is already used inside zifmia/sharpee but not hoisted; the root devDep makes it available to scripts/ without a package boundary.

### 3. Boot-Time Validation Gated on `useEngineRouter`
Many unit tests seed the scanner with fake paths. Running engine boots against those would fail every test. The `useEngineRouter: true` flag already gates production-style boot, so it is the right gate for production-style baseline validation.
