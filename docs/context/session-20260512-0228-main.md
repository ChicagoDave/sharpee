## Key Decisions

### 1. 6c scope upgraded to "Full" before implementation started

David chose Full ChannelService adoption over the MVP path at the start of 6c. This required splitting 6c into 6c-server (capture `channel:manifest`/`channel:packet` on the server) and 6c-client (wire renderers on the client) because Zifmia had never instantiated ChannelService at all. The split added ~80 tool calls but put the wire contract on the typed-packet shape ADR-163/165 prescribes — deferring this would have required a second wire cutover later.

### 2. Original 6f split into 6f-saves + 6f-admin

David's call to ship saves immediately and defer the admin dashboard to after 6g. This moved the multi-user E2E gate (6g) earlier and reduced the risk that 6g would run out of context budget.

### 3. `isAdmin` cached at WS connect, not per-subscribe

`isAdmin` is read once from the identity record at WS open and stamped onto presence frames. It is not refreshed when a room subscription is issued. This invariant is codified as an explicit test in `ws.test.ts` ("isAdmin does not change between subscribe calls for the same connection").

### 4. `--system` chat modifier deferred

ADR-176 vocabulary includes `--system` for server-emitted join/leave messages. The server currently emits no `chat:message` frames of its own; adding them requires touching WS emit paths. Deferred to a post-Phase-6 sub-phase. Tracked in plan open items.

### 5. Playwright over other E2E approaches

Playwright was chosen for 6g because: it satisfies rule 13a's "real browser" criterion literally; CI integration is a single `playwright install --with-deps chromium` step; the `@playwright/test` runner natively handles the `webServer` lifecycle. The ~50 MB devDep cost is acceptable given that Zifmia's primary value proposition is multi-user real-time correctness.

### 6. `RoomStub` retained after `main.ts` stopped routing through it

`RoomStub` and its 8 tests remain in place. They are not load-bearing after 6c but serve as a regression anchor while the room flow stabilizes. Removal planned for a post-6 cleanup commit.
