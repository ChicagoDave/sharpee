## Key Decisions

### 1. Events are records, not notifications
`report()` is a stdlib/command-executor phase that emits event records; the end-of-turn prose pipeline consumes those records via the language layer and text formatters. This is distinct from pub/sub. The term "Domain Event" must not appear in the book. Ch 3 now reflects this correctly.

### 2. No em dashes in book prose; colons for chapter titles
David is curating a global de-em-dash pass. All chapter titles use `Title: Subtitle` colon format. The per-chapter `edits-chNN.md` files carry the remaining prose-level proposals.

### 3. Two-pass editing model with platform verification gate
Mechanical em-dash pass runs first (low risk, no semantic change). Clarity pass runs second and any proposal that invents a concrete API example must be verified against platform code before adoption. Ch 19 caught the first violation of this rule.

### 4. Ch 28 self-containment via GitHub link
Decided in the parallel branch: Ch 28 multi-file story chapter resolves the "too much to show inline" problem with a link to the open-source GitHub repo. No further structural change needed.
