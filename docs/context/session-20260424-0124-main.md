- `GameEngine` + `engine.setStory(story)` + `engine.start()` (this last call was missing from the plan's bootstrap sketch — without it, `executeTurn` throws "Engine is not running"; found by reading Zifmia's `GameContext.tsx:393`)

Registers `ISaveRestoreHooks` as the bridge between the wire protocol and the engine:
- `onSaveRequested(data)` captures the `ISaveData` blob for subsequent SAVED wire emit
- `onRestoreRequested()` returns a slot populated by the RESTORE wire handler

Frame loop reads stdio JSON lines and dispatches on type:
- **INIT**: emits READY with real `story_metadata` from the embedded meta
- **COMMAND**: `engine.executeTurn(input)` inside try/catch; on success emits OUTPUT with `text_blocks` + `events`; on throw emits ERROR with `phase: 'turn'`
- **SAVE**: `engine.executeTurn('save')` (meta-command, does not advance turn counter) → platform-events path fires `platform.save_requested` → `ISaveRestoreHooks.onSaveRequested` captures blob → base64-encode (chunked `String.fromCharCode`, safe for arbitrary payload size) → emit SAVED
- **RESTORE**: base64-decode → stash in pending slot → `engine.executeTurn('restore')` → hook returns stash → engine loads via `SaveRestoreService.loadSaveData` + `EngineCallbacks` post-restore bookkeeping → emit RESTORED with `text_blocks: []`
- **SHUTDOWN**: emits EXITED `{ reason: 'shutdown' }`; `Deno.exit(0)`

`tsconfig.json` `exclude` narrowed from `src/sandbox/**` to `src/sandbox/deno-entry.ts` only — allows `story-bundler.ts` and `story-cache.ts` to compile into `dist/` while still excluding the Deno-only entry from the Node build.

## Key Decisions

### 1. No-Stub-Under-Test Enforced End-to-End

The acceptance gate, the bundler test, and the cache test all drive real esbuild, real Deno, real `@sharpee/*` graph, and real `dungeo.sharpee`. No stubs survive on the spawn path. The compound removal in Task 6 deliberately breaks the 28 legacy stub-backed tests as the concrete signal that drives 4-R-3 forward. Breakage by removal is the correct posture — it makes the debt visible rather than allowing it to continue masquerading as passing coverage.

### 2. Save/Restore via Platform Events (Course Correction Early in Session)

The plan sketch suggested calling engine internal methods directly. The user corrected this: "save and restore are done through platform events." Redirected to the hook-bridge pattern: the sandbox registers `ISaveRestoreHooks` and triggers save/restore via `engine.executeTurn('save' | 'restore')` (meta-commands). The engine's built-in `PlatformOperationHandler` wires hook callbacks to `SaveRestoreService` with full post-restore bookkeeping (`context.currentTurn`, parser pronoun reset, vocab update). No changes to any `packages/engine/` code were needed. Save blobs are engine-canonical `ISaveData`, serialized as base64-JSON; they travel sandbox → server → disk via wire SAVED frames, never to the browser (per user's guidance: "the container should store save files unlike the web/zifmia implementations").

### 3. `engine.start()` Required After `setStory`

Missing from the plan's bootstrap sketch. Zifmia's `GameContext.tsx:393` calls it. Without it, `executeTurn` throws "Engine is not running." Added after hook registration, before the first READY emit. This was caught when the acceptance gate initially reached 2/5 GREEN and the COMMAND test timed out — manual spawn + frame echo revealed the error.

### 4. Zero-Env Permission Posture Preserved via esbuild `define:`

The parser and stdlib graph reads `process.env.PARSER_DEBUG` and `process.env.DEBUG_PRONOUNS` at import time. Rather than grant `--allow-env` on spawn (which would widen the Deno permission surface), both vars were defined to `""` at bundle time via esbuild's `define:` option. The Deno sandbox never attempts an env read. ADR-153 Decision 1's narrow-permission intent preserved exactly.

### 5. Acceptance Gate Partial-Flip Diagnostic

After Task 6 + 7 landed, gate went from 0/5 → 2/5: INIT→READY and SHUTDOWN→EXITED passed immediately; COMMAND/SAVE/RESTORE each timed out at 10 s. Manual spawn with an echoed COMMAND frame revealed the Deno env-access denial (`Uncaught PermissionDenied: process.env`). The `define:` fix brought the gate to 4/5. The last failure (save/restore byte-identity assertion) was the warm-up `look` fix in Task 7. Final state: 5/5 GREEN.
