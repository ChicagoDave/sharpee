## Completed

### SessionStart Hook Bug Fix
The `.claude/hooks/session-start.sh` line 35 used a branch-scoped glob `session-*-${BRANCH}.md` to locate the most recent previous-session file. Session files retain the branch suffix from when they were created (e.g. `session-20260424-2329-lang-articles-migration.md`). After those branches merged to main, the hook silently skipped two substantive recent sessions and surfaced an older one instead.

Fix: changed the lookup to glob all `session-*.md` files with a defensive self-exclusion on today's in-flight file:
```sh
ls -t ${SESSION_DIR}/session-*.md 2>/dev/null | grep -v "/$(basename "$SESSION_FILE")$" | head -1
```
The other hooks (`session-update.sh`, `session-finalize.sh`) correctly stay branch-scoped — they need today's in-flight file specifically. Only the "what was done last time?" lookup needs to be unscoped.

Documented in `docs/work/devarch-session-start-bug.md`: symptom table, root cause, fix applied, and a recommendation that DevArch upstream adopt the unscoped lookup so the patch survives `devarch update`.

## Key Decisions

### 1. One-Shot Cutover vs. Two-Phase Rollout
Dropped the gradual migration path from ADR-159 before Phase 1 began. The server admin owns game lifecycle; participants must coordinate disruption at deploy time. This eliminates the "additive mode" complexity and the risk of running legacy and new paths in parallel for an extended period.

### 2. `findHashByUsername` as an Auth-Only Surface
The hash is never returned through the general `findByUsername` method — only through `findHashByUsername`. This is a structural enforcement of AC-6: code that needs an identity for display purposes cannot accidentally access the hash.

### 3. `deleted_at` Column Included in Phase 1
Rather than adding it in a later migration, `deleted_at` was included in the initial schema. This avoids a future `ALTER TABLE` and makes the admin-delete Phase 7 a script-only deliverable.

### 4. Stub HashService for Fast Tests, Real argon2 as the Acceptance Gate
`createStubHashService()` (deterministic `stub:<secret>`) keeps most tests sub-millisecond. Each test file that uses the stub names the real-argon2 test that backs it. The real-argon2 paths verify the DB row's `secret_hash` starts with `$argon2id$` and is not equal to the plaintext secret — satisfying Integration Reality for Phase 2.

### 5. Async Hello Without Breaking the Sync `on('message')` Shape
The WS `on('message')` handler stays sync-shaped; `handleHello` is wrapped in a fire-and-forget async IIFE inside the message handler. This avoids changing the server's overall message-dispatch contract while enabling `await` inside the hello path.

## Challenges Encountered

### Succession E2E Test Stale Token
`tests/ws/succession-e2e.test.ts:224` had a stray `openAndHello(alice.room_id, alice.token)` (legacy string token) left from the pre-cutover era. A `waitFor` timeout in a later assertion revealed the problem: the bearer token was passed to a function expecting `{username, secret}`, producing a malformed hello frame, which caused a WS close, which left the `waitFor` polling indefinitely. Diagnosed by isolating the timeout to the specific step and comparing the argument shape. Fixed inline.

### Validation Order Preservation in HTTP Routes
`create-room.ts` and `join-room.ts` had existing validation that checked specific fields in a specific order. Adding `identity_id` validation required care about where in the sequence it was inserted — placing it in the wrong spot caused existing tests' rejection-path assertions to fire on the new field instead of the one they expected. Resolved by reading the test expectations before placing the new validation.
