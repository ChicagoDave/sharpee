## Key Decisions

### 1. Option C — strip chapter numbers, don't renumber
Numbers were removed outright from nav, titles, in-page headings, link labels, and cross-references rather than kept and renumbered, since the numbering scheme itself was the source of drift across five independent layers.

### 2. Option A — monochrome navy palette
User disliked the rose/pink accent; the design generator's role mappings were switched to a navy scale end-to-end (link, wash, dark-mode button), with the semantic `warn` callout intentionally exempted.

### 3. Keep table cells and colon-lead-in convention
The prose-cleanup passes explicitly did not touch table cell contents or the "label:" lead-in style — these read correctly as-is and were out of scope for the em-dash/fragment fixes.

### 4. Verb/code labels stay lowercase
Nav Title Case applies only to structural group/section headers; leaf labels that are verbs or code identifiers (`taking and dropping`, `define phrase`) were kept lowercase to match in-prose usage.
