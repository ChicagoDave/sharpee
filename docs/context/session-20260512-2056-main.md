## Key Decisions

### 1. Symmetric submitter — no HTTP-body fallback for TurnPacket
`POST /command` returns `{turn_id}` only. `GET /state` is the recovery path for WS flakiness. Adding a TurnPacket in the HTTP body would reintroduce an asymmetric submitter path and implicitly make WS authoritative for non-submitters but optional for the submitter — the exact problem the design eliminates. Rationale codified in ADR-177 §3 invariants.

### 2. ifMUD / Floyd as counter-example, not contradiction
David clarified that ifMUD has integrated interpreters (stateful, runtime-coupled) and Floyd can only run one game at a time (singleton). Both are positioned as the stateful extreme on the curve. ADR-177's per-turn stateless durability is the distinguishing property vs. both ifMUD (singleton-stateful) and ADR-153 Deno (per-room-stateful).
